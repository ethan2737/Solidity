01 bytes 类型 — 动态字节数组

bytes 是 Solidity 中的动态字节数组，可以存储任意数量的字节。

  • 与 bytes1[] 不同，bytes 是连续内存，更节省 Gas
  • 支持 push() / pop() 动态增删元素
  • 索引访问返回 bytes1 类型

代码示例

contract BytesDemo { bytes public dynamicBytes; // 添加字节到末尾 function pushByte(uint8 _byte) public { dynamicBytes.push(bytes1(_byte)); } // 移除最后一个字节 function popByte() public { dynamicBytes.pop(); } // 获取指定索引字节 function getByte(uint256 _index) view returns(bytes1) { return dynamicBytes[_index]; } }

02 string 类型 — 本质就是 bytes

string 是 UTF-8 编码文本类型，本质上是 bytes 的封装。

⚠️ 为什么不能直接获取长度？

// 错误 - string 没有 length 属性 str.length; // 正确 - 需要先转成 bytes bytes(str).length;

字节长度 vs 字符长度

"你好" → bytes("你好").length = 6

中文字符在 UTF-8 中占 3 字节

字符个数 = bytes(str).length / 3

02 string 和 bytes 本质关系

string

只读的 bytes
需要转换才能操作

bytes

可读写的字节数组
可以直接操作

核心原则：所有字符串操作最终都是与 bytes 字节打交道

// 转换流程 string → bytes(str) → 操作 → string(bytes) // 获取长度 bytes(str).length // 访问字符 bytes(str)[0]

03 固定长度 bytes1-bytes32

bytes1 到 bytes32 是固定长度的字节数组类型，存储在 storage 时每个类型占用固定槽位。

常见用途

场景数据类型长度
哈希值bytes3232 字节 ✓
Token IDbytes3232 字节 ✓
签名bytes65 字节
bytes32 public fixedBytes32; // 32 字节 bytes1 public fixedBytes1; // 1 字节 bytes4 public fixedBytes4; // 4 字节 bytes20 public fixedBytes20; // 20 字节（地址）

03 ⚠️ bytes32 不是用来存字符串的！

bytes → bytes32 转换会丢失数据！

转换规则

// 长度 >= 32：截取前 32 字节（丢失数据！） // 长度 < 32：剩余部分填充零 // ✅ 短字符串（<= 32 字节）可以无损转换 string short = "Hi"; bytes32 b = bytes32(bytes(short)); // ❌ 长字符串（> 32 字节）会丢失数据！ string long = "这是一段很长的文字..."; bytes32 b = bytes32(bytes(long)); // 只保留前 32 字节！

正确做法：长字符串用 string 或 bytes 存储

04 bytes 与 string 互转

bytes 和 string 可以相互转换，转换是零成本的视图操作，不复制数据。

基础转换方法

// string → bytes（零成本） function stringToBytes(string memory _str) pure returns (bytes memory) { return bytes(_str); } // bytes → string（零成本） function bytesToString(bytes memory _bytes) pure returns (string memory) { return string(_bytes); }

其他常用转换

  • bytes32 → string — 通过 bytes 中转
  • uint256 → bytes — 32 字节
  • address → bytes — 20 字节

⚠️ 不同语言字符串长度对比

语言"你好".length说明
JavaScript2按字符计数
Python2按字符计数
Solidity6按字节计数（中=3字节）

💡 bytes("你好").length = 6，因为 UTF-8 中每个汉字占 3 字节

04 数值与地址的 bytes 转换

uint256 → bytes (32字节)

function uintToBytes(uint256 _value) pure returns (bytes memory) { bytes result = new bytes(32); for (uint256 i = 0; i < 32; i++) { result[31 - i] = bytes1(uint8(_value >> (i * 8))); } return result; }

address → bytes (20字节)

function addressToBytes(address _addr) pure returns (bytes memory) { bytes20 addrBytes = bytes20(_addr); bytes result = new bytes(20); for (uint256 i = 0; i < 20; i++) { result[i] = addrBytes[i]; } return result; }

bytes → uint256

function bytesToUint(bytes memory _bytes) pure returns (uint256) { require(_bytes.length >= 32, "Need 32 bytes"); uint256 result = 0; for (uint256 i = 0; i < 32; i++) { result = result * 256 + uint256(uint8(_bytes[i])); } return result; }

05 多维数组

Solidity 支持二维、三维甚至更高维度的数组，分为动态多维数组和固定长度多维数组。

声明方式

// 动态二维数组 - 行数可变，每行长度也可变 uint[][] public dynamic2DArray; // 固定长度二维数组 [3][5] = 5行 × 3列 uint[3][5] public fixed2DArray; // 混合：固定行数（3行），每行动态长度 uint[][3] public mixed2DArray; // 动态三维数组 uint[][][] public dynamic3DArray; // 固定三维数组 [2][3][4] = 4层×3行×2列 uint[2][3][4] public fixed3DArray;

05 ⚠️ 数组声明与访问的顺序规则

声明规则：从右往左看

uint[3][4][5]
= 5层×4行×3列

访问规则：从左往右写

arr[x][y][z]
= 第x层，第y行，第z列

示例

uint[2][3][4] public fixed3DArray; // 4层 × 3行 × 2列 // 设置值时按 层→行→列 顺序 set3DElement(uint layer, uint row, uint col, uint value) { fixed3DArray[layer][row][col] = value; }

记忆口诀：声明从右往左是 列→行→层，访问从左往右是 层→行→列

05 多维数组遍历示例

遍历二维数组计算总和

// 假设 dynamic2DArray = [[1,2,3], [4,5,6]] function sum2DArray() public view returns (uint256) { uint256 total = 0; // 外层循环：遍历每一行 for (uint256 i = 0; i < dynamic2DArray.length; i++) { // 内层循环：遍历当前行的每个元素 for (uint256 j = 0; j < dynamic2DArray[i].length; j++) { total += dynamic2DArray[i][j]; } } return total; // 返回 21 }

计算某一行的总和

function sumRow(uint256 _rowIndex) public view returns (uint256) { require(_rowIndex < dynamic2DArray.length, "Row not exist"); uint256 sum = 0; uint256[] memory row = dynamic2DArray[_rowIndex]; for (uint256 i = 0; i < row.length; i++) { sum += row[i]; } return sum; }

💡 遍历动态二维数组时，用 .length 获取行数和每行元素数

06 切片操作 — 没有内置语法

⚠️ Solidity 没有内置切片语法！

需要手动创建新数组并逐元素复制

对比其他语言

语言切片语法特点
Pythonarr[1:3]返回视图
JavaScriptslice(1,3)返回新数组
Solidity无内置语法需要手动复制

06 切片实现方式

通用切片函数

function getBytesSlice( uint256 _start, uint256 _length ) public view returns (bytes memory) { require(_start + _length <= dataBytes.length, "Out of bounds"); // 1. 创建新数组 bytes slice = new bytes(_length); // 2. 逐字节复制 for (uint256 i = 0; i < _length; i++) { slice[i] = dataBytes[_start + i]; } return slice; }

string 切片需要先转 bytes

function getStringSlice(uint256 _start, uint256 _length) public view returns (string memory) { bytes strBytes = bytes(dataString); // 先转 bytes // ... 同样的复制逻辑 ... return string(slice); }
