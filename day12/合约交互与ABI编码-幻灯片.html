Day12：合约交互与 ABI 编码

Solidity 高级编程

ABI Interface Call

什么是 ABI？

ABI（Application Binary Interface） = 合约的"使用说明书"

  • 有哪些函数可以调用
  • 函数的参数是什么类型
  • 返回值是什么类型
  • 事件怎么触发

API 程序员用的接口   VS   ABI 机器用的接口

为什么要"编码"？

EVM 只认识二进制！

人类写的代码：

setValue(100)

EVM 需要的数据：

0x60fe47b10000000000000000000000000000000000000000000000000000000064

abi.encode 就是把这个转换自动化了

两个 ABI 的区别 重点！

interface IABI

描述合约有哪些方法

interface IToken { function transfer(address to) external; }

abi.encode()

编解码二进制数据

bytes memory data = abi.encode(123); (uint256 v) = abi.decode(data, (uint256));

它们只是名字碰巧一样，实际上完全没关系！

接口定义 interface

接口 = 合约的"名片"，声明对方有什么方法

  • 只写函数签名，不写实现
  • 所有函数默认 external
  • 不能定义构造函数、状态变量
// 定义接口 interface IToken { function transfer(address to, uint256 amount) external; function balanceOf(address account) external view returns (uint256); }

函数选择器

函数选择器 = 方法名的"指纹"

是函数签名的 Keccak-256 哈希的前 4 字节

方法签名：setValue(uint256) ↓ Keccak256 哈希 完整哈希：0x60fe47b100000000...000064 ↓ 取前4字节 选择器： 0x60fe47b1 ← EVM识别方法

选择器 = 方法的"简短编号"，EVM通过它知道你要调用哪个方法

abi 编码方法对比

方法作用包含选择器补齐32字节
abi.encode标准参数编码否是
abi.decode解码还原数据否-
abi.encodePacked紧密打包否否
encodeWithSignature可执行的函数调用是是
encodeWithSelector手动选择器调用是是

encodeWithSignature = "带门牌号的快递"

选择器 = 地址，参数 = 货物

encodeWithSignature 重点

自动拼接：选择器 + 参数 = 可执行的函数调用

// 自动生成选择器 + 编码参数 bytes memory data = abi.encodeWithSignature("setValue(uint256)", 100); // 结果：0x60fe47b1 00000000000000000000000000000000000000000000000000000000000064 // ↑选择器 ↑参数

实际使用场景：用 call 调用合约

target.call(abi.encodeWithSignature("setValue(uint256)", 100));

call vs delegatecall 核心！

call

去别人地盘办事

  • 执行地点：目标合约
  • 修改：目标合约的数据
  • msg.sender = 调用合约

delegatecall

请别人来自己地盘

  • 执行地点：调用合约
  • 修改：调用合约的数据
  • msg.sender = 原始调用者

调用语法

两种调用都是通过目标合约地址来触发

address public target; // 目标合约地址 // 语法 target.call(data) target.delegatecall(data) // 示例 targetContract.call(abi.encodeWithSignature("setValue(uint256)", 100)); targetContract.delegatecall(abi.encodeWithSignature("setValue(uint256)", 100));

call：改别人的数据  |  delegatecall：改自己的数据

生活类比

call

委托外卖平台送餐

  • 告诉平台 → 商家做
  • 送到你家里
  • 改商家的库存

delegatecall

请隔壁老王做饭

  • 告诉老王 → 用老王菜谱
  • 在你家厨房做
  • 端上你家的桌

总结

知识点核心概念适用场景
ABI合约的"使用说明书"前端/后端调用合约
interface描述对方有什么方法合约间调用
abi.encode标准 ABI 编码参数编码、跨合约传递
call在目标合约上下文跨合约调用
delegatecall在调用合约上下文代理合约、升级模式

关键：通过"目标地址"来触发调用
