## 一、核心概念详解

### 1.1 什么是 Address（地址类型）？

#### 概念定义

`address` 是 Solidity 中专门用于存储**以太坊地址**的数据类型。以太坊地址是一个 **20 字节（160 位）** 的十六进制字符串，代表区块链上的一个账户或合约。

```solidity
// 地址示例（42个字符，包含0x前缀）
0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb5
```

#### 地址的两种类型

| 类型 | 定义 | 功能 | 使用场景 |
|------|------|------|----------|
| `address` | 普通地址类型 | 只能查询余额，**不能接收 ETH** | 普通账户、合约地址 |
| `address payable` | 可支付地址类型 | 可以查询余额，**可以接收 ETH** | 收款地址、资金库、需要转账的地址 |

#### 为什么需要区分？

Solidity 设计者为了**安全性**故意做了这个区分：

1. **防止意外转账**: 编译器会强制检查，确保只有标记为 `payable` 的地址才能接收 ETH
2. **明确意图**: 代码阅读者一眼就能看出哪些地址是用于收款的
3. **避免资金损失**: 防止 ETH 被误转到不支持收款的合约或地址

#### 地址类型完整代码示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract AddressExample {
    // ============ 地址变量声明 ============

    // 普通地址：只能存储地址，不能接收 ETH
    address public owner;
    address public userWallet;

    // payable 地址：可以接收 ETH
    address payable public treasury;      // 资金库
    address payable public paymentReceiver;  // 收款人

    // ============ 地址的基本操作 ============

    // 获取地址的 ETH 余额（任何地址都可以查余额）
    function getAddressBalance(address _addr) public view returns (uint256) {
        return _addr.balance;  // 返回 wei 单位的余额
    }

    // 获取合约自身的地址
    function getContractAddress() public view returns (address) {
        return address(this);  // this 指代当前合约实例
    }

    // 获取合约自身的 ETH 余额
    function getContractBalance() public view returns (uint256) {
        return address(this).balance;
    }

    // ============ 地址比较 ============

    // 检查两个地址是否相等
    function isSameAddress(address _a, address _b) public pure returns (bool) {
        return _a == _b;  // 地址比较是逐字节比较
    }

    // 检查地址是否为零地址
    function isZeroAddress(address _addr) public pure returns (bool) {
        // 零地址是默认值，表示"空"或"无效"
        return _addr == address(0);
    }

    // ============ 地址类型转换 ============

    // 普通地址 → payable 地址（需要显式转换）
    function toPayable(address _normalAddr) public pure returns (address payable) {
        // 使用 payable() 关键字进行转换
        return payable(_normalAddr);
    }

    // payable 地址 → 普通地址（隐式转换，自动完成）
    function fromPayable(address payable _payableAddr) public pure returns (address) {
        // 可以直接赋值，不需要显式转换
        return _payableAddr;
    }

    // ============ 转账操作（只有 payable 地址可以转账） ============

    // 向 payable 地址转账 ETH
    function sendETH(address payable _receiver, uint256 _amount) public {
        // transfer 方法：失败时自动 revert（回滚交易）
        _receiver.transfer(_amount);
    }
}
```

#### 关键知识点总结

1. **所有以太坊地址都是 20 字节**，与类型无关
2. **`address` 和 `address payable` 可以互相转换**：
   - `address` → `address payable`: 需要使用 `payable()` 显式转换
   - `address payable` → `address`: 可以直接赋值（隐式转换）
3. **只有 `address payable` 可以调用 `.transfer()` 和 `.send()` 方法**
4. **任何地址都可以查询 `.balance`** 属性

### 1.2 什么是 msg.sender？

#### 概念定义

`msg.sender` 是 Solidity 的**全局变量**，表示**当前函数调用的直接发起者**（一个地址）。

#### 重要特性

```solidity
// 特性1: msg.sender 的值会根据调用上下文变化
// 特性2: 在构造函数中，msg.sender 是合约的部署者
// 特性3: 在普通函数中，msg.sender 是调用该函数的账户或合约
// 特性4: 如果合约A调用合约B，那么在合约B中 msg.sender 是合约A的地址
```

#### 代码示例详解

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract MsgSenderExample {
    // 记录不同阶段的 msg.sender
    address public deployer;      // 部署者
    address public lastCaller;    // 最后调用者

    // ============ 构造函数 ============

    constructor() {
        // 部署合约时，msg.sender 是执行部署交易的账户地址
        deployer = msg.sender;
    }

    // ============ 普通函数 ============

    // 任何人都可以调用，记录调用者
    function doSomething() public {
        // 此时 msg.sender 是调用 doSomething() 的账户
        lastCaller = msg.sender;
    }

    // 只有部署者可以调用
    function onlyDeployerCanCall() public view returns (string memory) {
        // require 检查：如果不是部署者，交易会被拒绝
        require(msg.sender == deployer, "Only deployer can call this");
        return "Hello, deployer!";
    }

    // 获取当前调用者的地址
    function whoIsCalling() public view returns (address) {
        return msg.sender;
    }
}

// ============ 合约调用示例 ============

contract CallerContract {
    MsgSenderExample public target;  // 目标合约

    constructor(address _targetAddress) {
        // 通过地址引用其他合约
        target = MsgSenderExample(_targetAddress);
    }

    function callTarget() public {
        // 当这个函数被调用时：
        // - 在本合约中，msg.sender 是调用 callTarget() 的账户
        // - 在 target 合约中，msg.sender 是本合约的地址（CallerContract）
        target.doSomething();
    }
}
```

#### msg.sender 的权限控制模式

```solidity
// 经典的 Ownable（可拥有）模式
contract Ownable {
    address public owner;  // 记录合约所有者

    constructor() {
        // 部署时设置 owner 为部署者
        owner = msg.sender;
    }

    // 修饰符：限制只有 owner 可以调用
    modifier onlyOwner() {
        require(msg.sender == owner, "Not the owner");
        _;  // 这里会插入被修饰函数的代码
    }

    // 只有 owner 可以调用的函数
    function sensitiveFunction() public onlyOwner {
        // 敏感操作...
    }
}
```

### 1.3 什么是 Event（事件）？

#### 概念定义

Event（事件）是 Solidity 提供的一种**日志记录机制**。当合约执行过程中触发事件时，会将相关信息写入区块链的交易日志中。

#### 为什么需要事件？

```
┌─────────────────────────────────────────────────────────────┐
│                      事件的作用                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 省 Gas                                                  │
│     - 存储数据到合约状态变量：贵（永久存储）                  │
│     - 记录事件到交易日志：便宜（临时存储）                    │
│                                                             │
│  2. 方便外部监听                                            │
│     - DApp 可以实时订阅事件                                  │
│     - 钱包可以显示交易详情                                   │
│                                                             │
│  3. 可追溯历史                                              │
│     - 可以查询合约的所有历史操作                             │
│     - 方便审计和调试                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 事件定义与使用完整示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract EventExample {
    // ============ 事件定义 ============

    // 基础事件：记录转账
    // indexed 表示这个参数可以被索引搜索（最多3个indexed参数）
    event Transfer(
        address indexed from,    // 转出地址（可索引）
        address indexed to,      // 转入地址（可索引）
        uint256 amount           // 金额（不可索引）
    );

    // 复杂事件：记录状态变更
    event StatusChanged(
        address indexed user,    // 谁触发的
        uint256 indexed oldStatus,  // 旧状态
        uint256 indexed newStatus,  // 新状态
        string reason            // 变更原因
    );

    // 无参数事件：简单标记
    event ContractPaused();
    event ContractUnpaused();

    // ============ 事件触发 ============

    function transfer(address _to, uint256 _amount) public {
        // ... 执行转账逻辑 ...

        // 触发事件：emit 关键字 + 事件名 + 参数
        emit Transfer(msg.sender, _to, _amount);
    }

    function changeStatus(uint256 _newStatus, string memory _reason) public {
        uint256 oldStatus = 0;  // 假设查询到旧状态

        // ... 执行状态变更 ...

        // 触发事件
        emit StatusChanged(msg.sender, oldStatus, _newStatus, _reason);
    }

    function pause() public {
        // ... 暂停逻辑 ...
        emit ContractPaused();  // 无参数事件
    }
}
```

#### indexed 关键字详解

```solidity
// indexed 参数可以被外部快速搜索和过滤
// 一个事件最多有 3 个 indexed 参数

event Deposit(
    address indexed user,     // ✅ 可以搜索"某个用户的所有存款"
    uint256 indexed amount,   // ✅ 可以搜索"特定金额的所有存款"
    uint256 timestamp         // ❌ 没有 indexed，只能读取不能搜索
);

// 外部查询示例：
// - 查询 user = 0x123... 的所有 Deposit 事件 ✓
// - 查询 amount = 1000 的所有 Deposit 事件 ✓
// - 查询 timestamp > 1000000 的所有 Deposit 事件 ✗（需要遍历所有事件）
```

### 1.4 什么是 Constructor（构造函数）？

#### 概念定义

Constructor（构造函数）是合约中一个**特殊的函数**，它在合约**部署时自动执行一次**，用于初始化合约的状态变量。

#### 构造函数的特点

```solidity
// 特点1: 每个合约可以有 0 个或 1 个构造函数
// 特点2: 不写构造函数时，Solidity 自动生成一个空的
// 特点3: 构造函数只执行一次，之后无法再调用
// 特点4: 构造函数可以是 payable（用于工厂合约部署）
// 特点5: 构造函数中可以使用 msg.sender（部署者地址）
```

#### 构造函数完整示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract ConstructorExample {
    // ============ 状态变量 ============

    address public owner;           // 合约所有者
    address public manager;         // 管理员
    uint256 public creationTime;    // 创建时间
    string public name;             // 合约名称
    bool public initialized;        // 是否已初始化

    // ============ 构造函数 ============

    // 带参数的构造函数
    constructor(
        address _manager,      // 外部传入的管理员地址
        string memory _name    // 合约名称
    ) {
        // msg.sender: 部署此合约的账户地址
        owner = msg.sender;

        // 设置传入的参数
        manager = _manager;
        name = _name;

        // 记录创建时间
        creationTime = block.timestamp;

        // 标记已初始化
        initialized = true;
    }

    // ============ 使用场景 ============

    // 检查是否是 owner
    function isOwner(address _addr) public view returns (bool) {
        return _addr == owner;
    }
}

// ============ 无参构造函数示例 ============

contract SimpleContract {
    address public owner;

    // 无参数的构造函数
    constructor() {
        owner = msg.sender;
    }
}

// 等价于不写构造函数（Solidity 自动生成）：
// contract SimpleContract {
//     address public owner = msg.sender;  // 内联初始化
// }
```

#### 合约角色地址详解

```solidity
// 一个典型的合约可能有多种角色地址

contract RoleExample {
    // 部署者：创建合约的人（通常也是第一个 owner）
    address public deployer;

    // Owner：最高权限，可以修改关键配置
    address public owner;

    // Manager：管理权限，可以执行日常操作
    address public manager;

    // Treasury：资金库，接收和保管 ETH
    address payable public treasury;

    // Operator：操作员，可以执行特定功能
    address public operator;

    constructor(
        address _manager,
        address payable _treasury,
        address _operator
    ) {
        deployer = msg.sender;               // 记录部署者
        owner = msg.sender;                  // 部署者成为 owner
        manager = _manager;                  // 设置管理员
        treasury = _treasury;                // 设置资金库
        operator = _operator;                // 设置操作员
    }
}
```

### 1.5 什么是 Mapping（映射）？

#### 概念定义

Mapping（映射）是 Solidity 中的**键值对存储结构**，类似于其他语言中的字典（Dictionary）或哈希表（Hash Map）。

#### 映射的语法

```solidity
// 基本语法
mapping(KeyType => ValueType) public variableName;

// KeyType（键类型）：只能是基本类型（address, uint, bytes 等）
// ValueType（值类型）：可以是任何类型（包括数组、结构体、另一个映射）
```

#### 映射完整示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract MappingExample {
    // ============ 基础映射 ============

    // 地址 → bool：记录是否是成员
    // 用途：白名单、权限检查
    mapping(address => bool) public isMember;

    // 地址 → uint：记录余额
    // 用途：账户余额、积分、代币余额
    mapping(address => uint256) public balances;

    // ============ 嵌套映射 ============

    // 地址 → (地址 → uint)：记录对某个代币的授权额度
    // 用途：ERC20 的 approve 机制
    mapping(address => mapping(address => uint256)) public allowance;

    // 使用示例：
    // allowance[owner][spender] = 1000
    // 表示 owner 授权 spender 可以花费 1000 个代币

    // ============ 映射操作 ============

    // 添加/更新成员
    function addMember(address _user) public {
        isMember[_user] = true;  // 直接赋值，不存在则创建
    }

    // 查询成员
    function checkMember(address _user) public view returns (bool) {
        return isMember[_user];  // 如果不存在，返回默认值 false
    }

    // 增加余额
    function addBalance(address _user, uint256 _amount) public {
        balances[_user] += _amount;  // 直接累加
    }

    // 查询余额
    function getBalance(address _user) public view returns (uint256) {
        return balances[_user];  // 如果不存在，返回默认值 0
    }

    // 设置授权额度
    function approve(address _spender, uint256 _amount) public {
        // msg.sender 是授权方（owner）
        allowance[msg.sender][_spender] = _amount;
    }

    // 查询授权额度
    function getAllowance(address _owner, address _spender)
        public
        view
        returns (uint256)
    {
        return allowance[_owner][_spender];
    }
}
```

#### 映射的重要特性

```solidity
// 特性1: 所有可能的键都"存在"，默认值根据值类型而定
// - bool 默认值: false
// - uint 默认值: 0
// - address 默认值: address(0)

// 特性2: 无法遍历所有键（不知道哪些键被设置过）
// 解决方案：配合数组使用

contract MappingWithArray {
    mapping(address => uint256) public balances;
    address[] public users;  // 记录所有用户地址

    function addUser(address _user, uint256 _balance) public {
        // 如果是新用户，添加到数组
        if (balances[_user] == 0) {
            users.push(_user);
        }
        balances[_user] = _balance;
    }

    function getUserCount() public view returns (uint256) {
        return users.length;
    }
}
```

### 1.6 什么是 Contract Type（合约类型）？

#### 概念定义

在 Solidity 中，**每个合约定义本身就是一种类型**。就像 `uint`、`string` 是类型一样，`SimpleToken`、`AddressDemo` 这些合约名也是类型。

#### 核心理解

```solidity
// 类比理解：
// Java: UserService userService = new UserService();
//          ↑类型      ↑变量名

// Solidity: SimpleToken public token;
//              ↑类型      ↑变量名
```

#### 合约类型的两种使用方式

```solidity
// 方式1: 引用已部署的合约（通过地址）
SimpleToken token = SimpleToken(0x123...);

// 方式2: 部署新合约
SimpleToken newToken = new SimpleToken(1000);
```

#### 合约类型完整示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

// ============ 被引用的合约 ============

contract SimpleToken {
    string public name = "Simple Token";
    string public symbol = "ST";
    uint256 public totalSupply;
    mapping(address => uint256) public balanceOf;

    constructor(uint256 _initialSupply) {
        totalSupply = _initialSupply;
        balanceOf[msg.sender] = _initialSupply;
    }

    function transfer(address _to, uint256 _value) public returns (bool) {
        require(balanceOf[msg.sender] >= _value, "Insufficient balance");
        balanceOf[msg.sender] -= _value;
        balanceOf[_to] += _value;
        return true;
    }
}

// ============ 引用合约的合约 ============

import "./SimpleToken.sol";  // 导入合约定义

contract ContractTypeExample {
    // ============ 合约类型变量声明 ============

    // 声明一个 SimpleToken 类型的变量
    // 这就像说："我要存一个 SimpleToken 合约的引用"
    SimpleToken public token;

    // 也可以同时存地址
    address public tokenAddress;

    // ============ 设置合约引用 ============

    // 通过地址设置合约引用
    function setToken(address _tokenAddress) public {
        // 安全检查：不能是零地址
        require(_tokenAddress != address(0), "Invalid address");

        // 关键步骤：将 address 转换为合约类型
        // 这就像告诉编译器："相信我，这个地址上部署的是 SimpleToken"
        token = SimpleToken(_tokenAddress);

        // 同时保存地址
        tokenAddress = _tokenAddress;
    }

    // ============ 调用其他合约的函数 ============

    // 调用 token 合约的 name() 函数
    function getTokenName() public view returns (string memory) {
        // 先检查是否已设置
        require(address(token) != address(0), "Token not set");

        // 像调用本地函数一样调用其他合约的函数
        return token.name();
    }

    // 调用 token 合约的 balanceOf() 函数
    function getMyTokenBalance() public view returns (uint256) {
        // 查询当前调用者在 token 合约中的余额
        return token.balanceOf(msg.sender);
    }

    // 通过本合约转账 token
    function transferToken(address _to, uint256 _amount) public returns (bool) {
        // 注意：这里 msg.sender 是调用者
        // 但 token.transfer 中的 msg.sender 会变成本合约的地址！
        return token.transfer(_to, _amount);
    }

    // ============ 部署新合约 ============

    // 在本合约中部署新的 SimpleToken 合约
    function createNewToken(uint256 _initialSupply)
        public
        returns (address)
    {
        // 使用 new 关键字部署新合约
        // 这会创建一个新的合约实例，并返回合约地址
        SimpleToken newToken = new SimpleToken(_initialSupply);

        // 获取新合约的地址
        address newTokenAddress = address(newToken);

        // 可以自动设置为当前 token
        token = newToken;
        tokenAddress = newTokenAddress;

        return newTokenAddress;
    }

    // ============ 合约类型和地址的转换 ============

    // 合约类型 → 地址
    function getTokenAsAddress() public view returns (address) {
        return address(token);  // 直接转换
    }

    // 地址 → 合约类型
    function convertAddressToToken(address _addr)
        public
        pure
        returns (SimpleToken)
    {
        return SimpleToken(_addr);  // 显式转换
    }
}
```

#### 合约类型 vs 地址的关系

```
┌─────────────────────────────────────────────────────────────┐
│                    合约类型与地址的关系                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   SimpleToken.sol                                          │
│   ┌─────────────────────────────────────┐                  │
│   │ contract SimpleToken {              │                  │
│   │   function name() returns (string)  │ ← 类型定义       │
│   │   function transfer(...)            │                  │
```

### 1.7 全局变量详解（msg / block / tx）

#### msg 系列：当前消息调用信息

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract MsgVariables {
    // ============ msg.sender ============
    // 含义：当前函数调用的直接发送者（地址）
    // 使用场景：权限控制、记录操作人

    function getSender() public view returns (address) {
        return msg.sender;
    }

    // ============ msg.value ============
    // 含义：随本次调用发送的 ETH 数量（单位：wei）
    // 注意：函数必须标记为 payable 才能接收 ETH
    // 1 ETH = 10^18 wei

    function deposit() public payable {
        // msg.value 就是用户发送的 ETH 数量
        uint256 amount = msg.value;
        // ... 处理存款逻辑 ...
    }

    // ============ msg.data ============
    // 含义：完整的调用数据（calldata），包含函数选择器和参数
    // 使用场景：底层合约交互、代理合约

    function getCallData() public pure returns (bytes memory) {
        return msg.data;
    }

    // ============ msg.sig ============
    // 含义：函数调用的选择器（selector），即 msg.data 的前4字节
    // 计算方式：keccak256("functionName(paramTypes)") 的前4字节

    function getFunctionSelector() public pure returns (bytes4) {
        return msg.sig;
    }
    // 例如：调用 getSender() 时，msg.sig = 0x0a...（4字节）
}
```

#### block 系列：区块信息

```solidity
contract BlockVariables {
    // ============ block.timestamp ============
    // 含义：当前区块的时间戳（Unix 时间，单位：秒）

    // 含义：当前区块的编号（高度）

    uint256 public startBlock;
    uint256 public constant VOTING_PERIOD = 100;  // 100 个区块

    function startVoting() public {
```

#### tx 系列：交易信息

```solidity
contract TxVariables {
    // ============ tx.origin ============
    // 含义：整个交易链条的最初发起者
    // ⚠️ 重要警告：有安全风险，不要用做权限控制！
```

### 1.8 ETH 操作详解

#### 接收 ETH 的方式

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract ReceiveETH {
    mapping(address => uint256) public balances;

    // ============ receive() 函数 ============
    // 触发条件：用户直接发送 ETH 到合约地址，且没有附带任何调用数据
    // 特点：
    // - 必须是 external payable
    // - 不能有参数
    // - gas 限制 2300（不能执行复杂操作）

    receive() external payable {
        // 记录存款
        balances[msg.sender] += msg.value;
    }

    // ============ fallback() 函数 ============
    // 触发条件：调用不存在的函数，或发送 ETH 且附带数据

    fallback() external payable {
        // 可以做一些记录或拒绝操作
        balances[msg.sender] += msg.value;
    }

    // ============ payable 函数 ============
    // 显式定义接收 ETH 的函数

    function deposit() external payable {
        // 可以添加额外逻辑
        require(msg.value > 0, "Must send ETH");
        balances[msg.sender] += msg.value;
    }

    // 带额外参数的存款函数
    function depositWithNote(string memory _note) external payable {
        balances[msg.sender] += msg.value;
        // 可以记录备注（需要额外存储）
    }
}
```

#### 发送 ETH 的方式

```solidity
contract SendETH {
    // ============ 三种转账方式对比 ============

    // 方式1: transfer（推荐用于简单场景）
    // 特点：
    // - 固定 2300 gas，防止重入攻击
    // - 失败时自动 revert（回滚交易）
    // - 最安全的简单转账方式

    function sendViaTransfer(address payable _to, uint256 _amount) public {
        _to.transfer(_amount);  // 失败时自动回滚
    }

    // 方式2: send（不推荐）
    // 特点：
    // - 固定 2300 gas
    // - 返回 bool 表示成功/失败，需要手动检查

    function sendViaSend(address payable _to, uint256 _amount)
        public
        returns (bool)
    {
        bool success = _to.send(_amount);
        require(success, "Send failed");
        return success;
    }

    // 方式3: call（推荐用于高级场景）
    // 特点：
    // - 可自定义 gas 数量
    // - 返回 (bool, bytes memory)
    // - 可以调用接收合约的函数
    // ⚠️ 需要注意重入攻击防护！

    function sendViaCall(address payable _to, uint256 _amount) public {
        // {value: _amount} 指定发送的 ETH 数量
        // "" 表示不调用任何函数
        (bool success, ) = _to.call{value: _amount}("");
        require(success, "Call failed");
    }

    // 使用 call 调用接收合约的特定函数
    function callWithFunction(address _contract, uint256 _amount) public {
        // 编码函数调用：someFunction(param1, param2)
        bytes memory data = abi.encodeWithSignature(
            "someFunction(uint256)",
            123
        );

        (bool success, bytes memory returnData) = _contract.call{value: _amount}(data);
        require(success, "Call failed");
    }
}
```

#### 安全的取款模式（Checks-Effects-Interactions）

```solidity
contract SecureWithdrawal {
    mapping(address => uint256) public balances;

    // ============ 安全原则：CEI 模式 ============
    // C - Checks（检查）
    // E - Effects（更新状态）
    // I - Interactions（外部交互）
    // 这是防止重入攻击的标准模式

    function withdraw(uint256 _amount) public {
        // ===== 1. Checks（检查条件） =====
        // 先检查用户是否有足够的余额
        require(balances[msg.sender] >= _amount, "Insufficient balance");

        // 检查合约是否有足够的 ETH
        require(address(this).balance >= _amount, "Contract has no ETH");

        // ===== 2. Effects（更新状态） =====
        // 先扣减余额（在转账之前！）
        balances[msg.sender] -= _amount;

        // ===== 3. Interactions（外部交互） =====
        // 最后才进行转账操作
        (bool success, ) = payable(msg.sender).call{value: _amount}("");
        require(success, "Transfer failed");

        // 为什么这个顺序很重要？
        // 如果先转账再扣减余额，恶意合约可以在收到 ETH 时
        // 再次调用 withdraw()，形成"重入攻击"，多次取款
    }

    // ============ 重入攻击防护修饰符 ============

    bool private locked;  // 锁标志

    modifier nonReentrant() {
        require(!locked, "Reentrant call");
        locked = true;
        _;  // 执行被修饰的函数
        locked = false;
    }

    function withdrawWithProtection(uint256 _amount) public nonReentrant {
        require(balances[msg.sender] >= _amount, "Insufficient balance");
        balances[msg.sender] -= _amount;
        payable(msg.sender).transfer(_amount);
    }
}
```

### Q2: 什么是 mapping？`mapping(address => bool)` 如何使用？

**详细解答**：

`mapping` 是 Solidity 中的键值对存储结构，语法为 `mapping(KeyType => ValueType)`。

```solidity
// 定义：地址 → 布尔值的映射
mapping(address => bool) public isMember;

// 含义解释：
// - KeyType: address（键，用于查询）
// - ValueType: bool（值，存储的数据）
// - 整体含义：给定一个地址，查询它是否是成员（true/false）

// 使用示例：
function addMember(address _user) public {
    isMember[_user] = true;  // 设置键值对
}

function checkMember(address _user) public view returns (bool) {
    return isMember[_user];  // 查询键对应的值
    // 如果从未设置过，返回默认值 false
}
```

**类比理解**：
- 就像 Excel 表格：A列是地址，B列是布尔值
- 或者 Python 字典：`{"0x123...": true, "0x456...": false}`

### Q3: Event 事件到底是什么？

**详细解答**：

Event 是智能合约的"日志广播系统"。当合约中发生重要操作时，通过 event 记录到区块链上，外部应用可以监听这些事件。

```solidity
// 定义事件
event Transfer(address indexed from, address indexed to, uint256 amount);

// 触发事件
function transfer(address _to, uint256 _amount) public {
    // ... 执行转账逻辑 ...
    emit Transfer(msg.sender, _to, _amount);  // 广播事件
}
```

**为什么需要 event？**

1. **省 Gas**：存储到状态变量很贵，记录事件很便宜
2. **可监听**：DApp 可以实时订阅特定事件
3. **可追溯**：可以查询合约的所有历史事件

**类比**：银行的短信通知服务
- 存款/取款 → 银行发送短信通知
- Event → 合约发出事件通知

### Q4: 构造函数 constructor 详解

**详细解答**：

Constructor 是合约的"初始化函数"，在部署时自动执行一次。

```solidity
contract Example {
    address public owner;
    uint256 public createdAt;

    // 构造函数：部署时自动执行
    constructor() {
        owner = msg.sender;           // 记录部署者
        createdAt = block.timestamp;  // 记录创建时间
    }
}
```

**关键特性**：
- 只执行一次（部署时）
- 不能有返回值
- 可以是 payable（用于工厂模式）
- 不写时 Solidity 自动生成空的

**部署流程**：
```
1. 编写合约代码
2. 编译合约
3. 发送部署交易 → 构造函数执行
4. 合约地址生成
5. 部署完成，构造函数无法再调用
```

### Q5: address 和 address payable 的本质区别

**详细解答**：

**技术本质**：两者都是 20 字节的以太坊地址，区别在于**编译器允许的操作**不同。

```solidity
// address：只能查询，不能转账
address normalAddr = 0x123...;
normalAddr.balance;              // ✅ 可以查询余额
// normalAddr.transfer(1 ether); // ❌ 编译错误！普通地址不能转账

// address payable：可以查询，也可以转账
address payable payableAddr = payable(0x123...);
payableAddr.balance;             // ✅ 可以查询余额
payableAddr.transfer(1 ether);   // ✅ 可以转账
```

**设计原因**：
Solidity 设计者为了安全，强制开发者显式标记哪些地址是"可收款的"，防止意外转账。

**转换规则**：
```solidity
// address → address payable（需要显式转换）
address payable p = payable(normalAddr);

// address payable → address（自动转换）
address n = payableAddr;  // 直接赋值即可
```

### Q6: 数组操作 push / length / memory 详解

**详细解答**：

```solidity
contract ArrayExample {
    address[] public members;  // 动态数组

    // push：在数组末尾添加元素
    function addMember(address _member) public {
        members.push(_member);  // 数组长度 +1
    }

    }
```

**memory vs storage**：

- `memory`：临时存储在内存中（函数参数和返回值）

### Q7: 如何安全地接收和发送 ETH？

**详细解答**：

**接收 ETH**：
```solidity
// 方式1：receive 回调函数
receive() external payable {
    balances[msg.sender] += msg.value;
}

// 方式2：payable 函数
function deposit() external payable {
    require(msg.value > 0, "Must send ETH");
    balances[msg.sender] += msg.value;
}
```

**发送 ETH（安全模式）**：
```solidity
function withdraw(uint256 _amount) public {
    // 1. 检查
    require(balances[msg.sender] >= _amount, "Insufficient");

    // 2. 更新（先扣款！）
    balances[msg.sender] -= _amount;

    // 3. 转账（后转账！）
    payable(msg.sender).transfer(_amount);
}
```

**为什么要"先扣款后转账"？**
防止"重入攻击"——恶意合约在收到 ETH 时自动再次调用 withdraw，如果先转账，余额还没扣，就可以多次取款。

### Q8: changeOwner 和 address(0) 详解

**changeOwner**：转移合约所有权

```solidity
function changeOwner(address _newOwner) public {
```

```solidity
address(0) = 0x0000000000000000000000000000000000000000
```

- 零地址是默认值（相当于 null）
- 转账到零地址会永久丢失 ETH
- 必须检查 `_addr != address(0)`

```solidity
block.timestamp   // 区块时间戳（秒）

block.coinbase    // 矿工地址
block.prevrandao  // 随机数（不安全）
```

## 四、代码模板

### 模板1：完整的 Ownable 权限控制合约

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

/**
 * @title Ownable
 * @dev 提供基本的权限控制功能，只有一个 owner 可以执行敏感操作
 */
contract Ownable {
    // ============ 状态变量 ============

    // 合约所有者的地址
    address private _owner;

    // ============ 事件 ============

    // 当所有权转移时触发
    event OwnershipTransferred(
        address indexed previousOwner,
        address indexed newOwner
    );

    // ============ 修饰符 ============

    /**
     * @dev 限制只有 owner 可以调用
     */
    modifier onlyOwner() {
        require(
            msg.sender == _owner,
            "Ownable: caller is not the owner"
        );
        _;
    }

    // ============ 构造函数 ============

    /**
     * @dev 初始化合约，将部署者设为 owner
     */
    constructor() {
        _owner = msg.sender;
        emit OwnershipTransferred(address(0), msg.sender);
    }

    // ============ 查询函数 ============

    /**
     * @dev 返回当前 owner 的地址
     */
    function owner() public view returns (address) {
        return _owner;
    }

    // ============ 管理函数 ============

    /**
     * @dev 转移所有权到新地址
     * @param newOwner 新的 owner 地址
     * 要求：
     * - 只有当前 owner 可以调用
     * - newOwner 不能是零地址
     */
    function transferOwnership(address newOwner) public onlyOwner {
        require(
            newOwner != address(0),
            "Ownable: new owner is the zero address"
        );

        address oldOwner = _owner;
        _owner = newOwner;

        emit OwnershipTransferred(oldOwner, newOwner);
    }

    /**
     * @dev 放弃所有权（谨慎使用！）
     * 之后合约将没有 owner，所有 onlyOwner 函数都无法调用
     */
    function renounceOwnership() public onlyOwner {
        address oldOwner = _owner;
        _owner = address(0);
        emit OwnershipTransferred(oldOwner, address(0));
    }
}
```

### 模板2：安全的 ETH 存取合约

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

/**
 * @title ETHBank
 * @dev 安全的 ETH 存取合约，使用 Checks-Effects-Interactions 模式
 */
contract ETHBank {
    // ============ 状态变量 ============

    // 记录每个地址的存款余额
    mapping(address => uint256) private _balances;

    // 重入锁
    bool private _locked;

    // ============ 事件 ============

    event Deposit(address indexed account, uint256 amount);
    event Withdrawal(address indexed account, uint256 amount);

    // ============ 修饰符 ============

    /**
     * @dev 防止重入攻击的修饰符
     */
    modifier nonReentrant() {
        require(!_locked, "Reentrant call");
        _locked = true;
        _;
        _locked = false;
    }

    // ============ 接收 ETH ============

    /**
     * @dev 接收直接转账的 ETH
     * 当用户直接发送 ETH 到合约地址时触发
     */
    receive() external payable {
        _deposit();
    }

    /**
     * @dev 显式存款函数
     */
    function deposit() external payable {
        _deposit();
    }

    /**
     * @dev 内部存款逻辑
     */
    function _deposit() internal {
        require(msg.value > 0, "Must send ETH");

        _balances[msg.sender] += msg.value;

        emit Deposit(msg.sender, msg.value);
    }

    // ============ 查询函数 ============

    /**
     * @dev 查询指定地址的存款余额
     */
    function balanceOf(address account) public view returns (uint256) {
        return _balances[account];
    }

    /**
     * @dev 查询当前调用者的存款余额
     */
    function myBalance() public view returns (uint256) {
        return _balances[msg.sender];
    }

    /**
     * @dev 查询合约总余额
     */
    function totalBalance() public view returns (uint256) {
        return address(this).balance;
    }

    // ============ 取款函数 ============

    /**
     * @dev 取款指定金额
     * @param amount 要取款的金额（wei）
     * 使用 Checks-Effects-Interactions 模式防止重入攻击
     */
    function withdraw(uint256 amount) external nonReentrant {
        // ===== Checks（检查条件）=====
        require(amount > 0, "Amount must be positive");
        require(_balances[msg.sender] >= amount, "Insufficient balance");
        require(
            address(this).balance >= amount,
            "Contract has insufficient ETH"
        );

        // ===== Effects（更新状态）=====
        // 先扣减余额，防止重入攻击
        _balances[msg.sender] -= amount;

        // ===== Interactions（外部调用）=====
        // 最后才进行转账
        (bool success, ) = payable(msg.sender).call{value: amount}("");
        require(success, "Transfer failed");

        emit Withdrawal(msg.sender, amount);
    }

    /**
     * @dev 取出全部存款
     */
    function withdrawAll() external nonReentrant {
        uint256 amount = _balances[msg.sender];
        require(amount > 0, "No balance to withdraw");

        _balances[msg.sender] = 0;

        (bool success, ) = payable(msg.sender).call{value: amount}("");
        require(success, "Transfer failed");

        emit Withdrawal(msg.sender, amount);
    }
}
```

### 模板3：合约交互示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

// ============ 接口定义 ============

/**
 * @dev ERC20 代币接口
 * 只声明我们需要调用的函数
 */
interface IERC20 {
    function name() external view returns (string memory);
    function symbol() external view returns (string memory);
    function decimals() external view returns (uint8);
    function totalSupply() external view returns (uint256);
    function balanceOf(address account) external view returns (uint256);
    function transfer(address to, uint256 amount) external returns (bool);
    function allowance(address owner, address spender) external view returns (uint256);
    function approve(address spender, uint256 amount) external returns (bool);
    function transferFrom(address from, address to, uint256 amount) external returns (bool);

    event Transfer(address indexed from, address indexed to, uint256 value);
    event Approval(address indexed owner, address indexed spender, uint256 value);
}

// ============ 主合约 ============

/**
 * @title TokenManager
 * @dev 管理代币合约引用的示例
 */
contract TokenManager {
    // ============ 状态变量 ============

    // 代币合约引用
    IERC20 public token;

    // 记录代币合约地址
    address public tokenAddress;

    // 记录已授权的代币额度
    mapping(address => uint256) public approvedAmount;

    // ============ 事件 ============

    event TokenSet(address indexed tokenAddress);
    event TokenDeposited(address indexed user, uint256 amount);
    event TokenWithdrawn(address indexed user, uint256 amount);

    // ============ 构造函数 ============

    constructor() {
        // 初始时 token 为零地址
        tokenAddress = address(0);
    }

    // ============ 设置代币合约 ============

    /**
     * @dev 设置代币合约地址
     * @param _tokenAddress 代币合约的地址
     */
    function setToken(address _tokenAddress) external {
        require(_tokenAddress != address(0), "Invalid token address");

        // 地址 → 合约类型转换
        token = IERC20(_tokenAddress);
        tokenAddress = _tokenAddress;

        emit TokenSet(_tokenAddress);
    }

    /**
     * @dev 检查是否已设置代币合约
     */
    modifier tokenSet() {
        require(tokenAddress != address(0), "Token not set");
        _;
    }

    // ============ 查询函数 ============

    /**
     * @dev 获取代币名称
     */
    function getTokenName() external view tokenSet returns (string memory) {
        return token.name();
    }

    /**
     * @dev 获取代币符号
     */
    function getTokenSymbol() external view tokenSet returns (string memory) {
        return token.symbol();
    }

    /**
     * @dev 获取代币总供应量
     */
    function getTotalSupply() external view tokenSet returns (uint256) {
        return token.totalSupply();
    }

    /**
     * @dev 获取指定地址的代币余额
     */
    function getTokenBalance(address account)
        external
        view
        tokenSet
        returns (uint256)
    {
        return token.balanceOf(account);
    }

    /**
     * @dev 获取本合约的代币余额
     */
    function getContractTokenBalance()
        external
        view
        tokenSet
        returns (uint256)
    {
        return token.balanceOf(address(this));
    }

    // ============ 代币操作 ============

    /**
     * @dev 存入代币到本合约
     * @param amount 存入数量
     * 调用者需要先 approve 本合约才能转账
     */
    function depositTokens(uint256 amount) external tokenSet {
        require(amount > 0, "Amount must be positive");

        // 从调用者转账到本合约
        // 注意：这里 msg.sender 是调用者
        bool success = token.transferFrom(msg.sender, address(this), amount);
        require(success, "Transfer failed");

        // 记录存入金额
        approvedAmount[msg.sender] += amount;

        emit TokenDeposited(msg.sender, amount);
    }

    /**
     * @dev 从本合约提取代币
     * @param amount 提取数量
     */
    function withdrawTokens(uint256 amount) external tokenSet {
        require(amount > 0, "Amount must be positive");
        require(approvedAmount[msg.sender] >= amount, "Insufficient deposited");

        // 先更新状态
        approvedAmount[msg.sender] -= amount;

        // 再转账
        bool success = token.transfer(msg.sender, amount);
        require(success, "Transfer failed");

        emit TokenWithdrawn(msg.sender, amount);
    }
}
```

## 五、常见错误

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `EACCES: permission denied 127.0.0.1:8545` | Windows 端口占用 | 更换端口或重启 winnat 服务 |
| `Function is not payable` | 函数未标记 payable 但接收了 ETH | 添加 `payable` 关键字 |
| `Invalid address` | 使用了 address(0) | 检查 `require(_addr != address(0))` |
| `transfer not available` | 普通 address 调用 transfer | 转换为 payable：`payable(addr).transfer()` |
| `Member "name" not found` | 合约类型不匹配 | 确认导入的合约接口正确 |
| `Reentrant call` | 重入攻击被阻止 | 使用 Checks-Effects-Interactions 模式 |
| `Insufficient balance` | 余额不足 | 检查余额和授权额度 |

## 六、安全实践

### ✅ 应该做的

```solidity
// 1. 使用 msg.sender 做权限控制
require(msg.sender == owner, "Not owner");

// 2. 检查零地址
require(_addr != address(0), "Zero address");

// 3. Checks-Effects-Interactions 模式
function withdraw(uint256 amount) public {
    require(balances[msg.sender] >= amount);  // Checks
    balances[msg.sender] -= amount;            // Effects
    payable(msg.sender).transfer(amount);      // Interactions
}

// 4. 使用 transfer 或带保护的 call
// transfer 限制 2300 gas，防止重入
payable(addr).transfer(amount);

// 或使用非重入修饰符 + call
(bool success, ) = payable(addr).call{value: amount}("");
require(success);
```

### ❌ 不应该做的

```solidity
// 1. 不要用 tx.origin
require(tx.origin == owner);  // 危险！可被钓鱼攻击

// 2. 不要先转账后更新
payable(msg.sender).transfer(amount);  // ❌
balances[msg.sender] -= amount;        // 可能被重入

// 3. 不要依赖 block.timestamp 做随机
uint256 random = block.timestamp % 10;  // 可被矿工操控

// 4. 不要忘记检查返回值
someAddress.send(amount);  // ❌ 忽略返回值
```
