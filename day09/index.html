什么是 Modifier？

函数修改器（Modifier）

Solidity 中用于修改函数行为的特殊代码块

可以在函数执行前或后自动执行特定的检查逻辑

  • 代码复用 — 避免重复的验证逻辑
  • 访问控制 — 限制谁可以调用某些函数
  • 状态检查 — 在特定状态下允许或禁止操作

Modifier 基本结构

1modifier modifierName() {
2 // 前置检查逻辑
3 require(...);
4 _; // 函数体执行位置（关键！）
5 // 后置逻辑（可选）
6}

_; 是占位符，表示被修饰的函数体执行的位置

Modifier 执行顺序

1modifier modifierA() {
2 // 前置逻辑 A
3 _;
4 // 后置逻辑 A
5}
6
7modifier modifierB() {
8 // 前置逻辑 B
9 _;
10 // 后置逻辑 B
11}

执行顺序：modifierA 前置 → modifierB 前置 → 函数体 → modifierB 后置 → modifierA 后置

Modifier 继承

继承父合约 Modifier

1modifier onlyOwner() virtual {
2 require(msg.sender == owner);
3 _;
4}

使用 virtual 使其可被覆盖

覆盖 Modifier

1modifier onlyOwner() override {
2 require(msg.sender == owner ||
3 msg.sender == admin);
4 _;
5}

使用 override 覆盖父合约

命名规范

使用 NatSpec 注释（@notice、@dev）描述用途

错误消息

错误消息要清晰，便于调试和用户理解

virtual 关键字

如果需要被继承覆盖，使用 virtual 声明

常见错误 & 注意事项

场景 说明
忘记使用 _; Modifier 必须包含 _; 否则函数体不会执行
Modifier 顺序错误 多个 Modifier 时顺序很重要，从左到右依次执行
覆盖忘加 override Solidity 0.8+ 需要显式声明 override
在 Modifier 中修改状态 谨慎处理，避免重入攻击等安全问题

总结

Modifier 的核心价值

  • 提高代码可读性 — 将验证逻辑从函数主体中分离
  • 增强安全性 — 确保关键函数只被授权方调用
  • 促进代码复用 — 避免重复编写相同的检查逻辑
  • 支持继承扩展 — 通过 virtual 和 override 实现灵活扩展

掌握 Modifier 是编写安全可靠的 Solidity 智能合约的基础
