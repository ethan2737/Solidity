什么是流动性挖矿？

// 用户视角 质押代币 → 获得奖励代币 // 项目方视角 获取用户资产 → 分发项目代币

双赢：用户获得收益，项目获得流动性

核心概念

质押 (Stake) 用户将代币存入合约，锁定
解除质押 (Unstake) 用户取出质押的代币
奖励代币 项目方发放的奖励代币
质押代币 用户需要存入的代币 (USDT/ETH)
奖励速率 每秒/每区块发放的奖励数量
精度因子 1e18 避免整数除法导致结果为0

合约整体架构

contract StakingMining { // 两种代币 IERC20 public stakingToken; // 质押代币 IERC20 public rewardToken; // 奖励代币 // 权限 address public owner; bool public paused; // 奖励系统 uint256 public rewardRate; // 每秒奖励 uint256 public periodFinish; // 奖励结束时间 uint256 public rewardPerTokenStored; // 累计每token奖励 uint256 public lastUpdateTime; // 用户数据 struct UserInfo { uint256 stakedAmount; // 质押数量 uint256 rewardPerTokenPaid; // 已付每token奖励 uint256 rewards; // 待领取奖励 uint256 lastStakeTime; // 上次质押时间 } mapping(address => UserInfo) public userinfo; uint256 public totalStaked; }

核心函数

stake()

质押代币到合约

unstake()

解除质押，取回代币

claimReward()

领取已赚取的奖励

质押函数 stake()

function stake(uint256 amount) external { // 1. Checks - 检查 if (amount == 0) revert InvalidAmount(); if (stakingToken.balanceOf(msg.sender) < amount) revert InsufficientBalance(); uint256 balanceBefore = stakingToken.balanceOf(address(this)); // 2. Effects - 更新状态 (先于外部调用!) UserInfo storage user = userinfo[msg.sender]; user.stakedAmount += amount; user.lastStakeTime = block.timestamp; totalStaked += amount; // 3. Interactions - 外部调用 (最后执行) stakingToken.transferFrom(msg.sender, address(this), amount); // 验证余额变化 uint256 balanceAfter = stakingToken.balanceOf(address(this)); if (balanceAfter != balanceBefore + amount) { revert FlashLoanAttackDetected(); } }

奖励计算公式

// 每质押1个代币，累计应得多少奖励 function rewardPerToken() public view returns (uint256) { if (totalStaked == 0) return rewardPerTokenStored; uint256 lastTime = lastTimeRewardApplicable(); uint256 timeElapsed = lastTime - lastUpdateTime; // 公式: 时间差 × 速率 × 1e18 ÷ 总质押量 uint256 reward = timeElapsed * rewardRate * 1e18 / totalStaked; return rewardPerTokenStored + reward; }

1e18 精度因子：避免整数除法导致结果为0

lastTimeRewardApplicable()

// 获取奖励适用的最后时间 function lastTimeRewardApplicable() public view returns (uint256) { return block.timestamp < periodFinish ? block.timestamp : periodFinish; }
  • 返回 min(当前时间, 结束时间)
  • 防止奖励无限累积
  • 确保奖励发放不超过设定周期

计算用户应得奖励

// 用户当前可领取多少奖励 function earned(address account) public view returns (uint256) { UserInfo storage user = userinfo[account]; if (totalStaked == 0) return user.rewards; // 新增奖励 = 质押数量 × 每token增量 ÷ 1e18 uint256 currentRewardPerToken = rewardPerToken() - user.rewardPerTokenPaid; uint256 newRewards = user.stakedAmount * currentRewardPerToken / 1e18; return user.rewards + newRewards; }

updateReward 修改器

modifier updateReward(address account) { // 1. 更新全局累计奖励 rewardPerTokenStored = rewardPerToken(); // 2. 更新最后时间 lastUpdateTime = lastTimeRewardApplicable(); // 3. 更新用户奖励记录 if (account != address(0)) { userinfo[account].rewards = earned(account); userinfo[account].rewardPerTokenPaid = rewardPerTokenStored; } // 4. 执行函数体 _; }

每次质押/解除质押/领取奖励时自动更新用户奖励

CEI 安全模式

错误顺序 ❌

  • 先转账
  • 后更新状态
  • → 可被重入攻击

正确顺序 ✓

  • 1. Checks 检查
  • 2. Effects 更新状态
  • 3. Interactions 外部调用

防止攻击者通过回调函数重新进入合约

闪电贷防护

⚠️ 常见误区：余额检查 ≠ 防范闪电贷

余额检查的真正作用：

  • 防止代币合约回调钩子导致余额异常

真正的闪电贷防护：

  • 1. 最短质押期限：要求质押满一定时间才能领取奖励
  • 2. CEI模式：防止重入
  • 3. 区块时间戳：闪电贷无法操控时间
  • 4. 奖励延迟释放：奖励不是立即可领取

暂停机制

// 暂停修改器 modifier whenNotPaused() { if (paused) revert ContractPausedError(); _; } // 所有关键函数都加上这个修改器 function stake(...) external whenNotPaused { ... } function unstake(...) external whenNotPaused { ... } function claimReward(...) external whenNotPaused { ... }
  • 发现合约bug → 暂停保护用户资金
  • 发现异常攻击 → 暂停阻止攻击
  • 项目升级 → 暂停过渡
