第一部分

接口 Interface

定义合约标准，实现代码解耦

接口概念介绍

接口（Interface）是 Solidity 中用于定义合约标准的重要机制。接口只定义函数签名，不包含实现代码，不能有状态变量。

通过接口可以实现：

  • 代码复用 — 标准化设计模式
  • 标准化设计 — 如 ERC20、ERC721 标准
  • 合约间解耦 — 降低合约间依赖

接口使用方法

定义规范

  • 使用 interface 关键字
  • 函数必须是 external 可见性
  • 不能包含函数实现
  • 不能有状态变量

实现规范

  • 合约使用 is 实现接口
  • 必须实现所有函数
  • 使用 override 关键字
  • 可继承多个接口

接口代码示例

interface IERC20Like { // 接口函数必须是 external 可见性 function totalSupply() external view returns (uint256); function balanceOf(address account) external view returns (uint256); function transfer(address to, uint256 amount) external returns (bool); function transferFrom(address from, address to, uint256 amount) external returns (bool); function approve(address spender, uint256 amount) external returns (bool); function allowance(address owner, address spender) external view returns (uint256); // 接口可以包含事件定义 event Transfer(address indexed from, address indexed to, uint256 value); event Approval(address indexed owner, address indexed spender, uint256 value); }

接口实现示例

contract SimpleToken is IERC20Like { mapping(address => uint256) private _balances; mapping(address => mapping(address => uint256)) private _allowances; uint256 private _totalSupply; // 实现接口函数必须使用 override 关键字 function totalSupply() external view override returns (uint256) { return _totalSupply; } function transfer(address to, uint256 amount) external override returns (bool) { require(_balances[msg.sender] >= amount, "Insufficient balance"); _balances[msg.sender] -= amount; _balances[to] += amount; emit Transfer(msg.sender, to, amount); return true; } }

接口调用其他合约

contract InterfaceUser { // 通过接口调用存储合约的 store 函数 function storeViaInterface(address storageContract, uint256 value) external { IStorable(storageContract).store(value); } // 通过接口读取存储合约的值 function retrieveViaInterface(address storageContract) external view returns (uint256) { return IStorable(storageContract).retrieve(); } // try/catch 捕获外部合约调用异常 function checkInterface(address contractAddress) external view returns (bool) { try IStorable(contractAddress).retrieve() returns (uint256) { return true; } catch { return false; } } }

接口核心要点

  • 接口 = 外部约束 — 定义合约必须有哪些方法，规范合约标准
  • 接口调用其他合约 — 通过接口与已部署合约交互
  • 接口 vs 实现 — 接口定义"做什么"，实现合约定义"怎么做"
  • try/catch 作用 — 捕获外部合约调用的异常

注意事项

  • 接口函数必须是 external
  • 不能包含实现和状态变量
  • 实现接口的合约必须实现所有函数
  • 命名规范：接口名以 "I" 开头，如 IERC20
第二部分

继承 Inheritance

实现代码复用和功能扩展

继承概念介绍

继承（Inheritance）是 Solidity 中实现代码复用的重要机制。通过继承，子合约可以复用父合约的状态变量、函数和修饰符。Solidity 支持单继承和多继承。

单继承

一个合约继承一个合约

contract Child is Parent

多继承

一个合约继承多个合约，用逗号分隔

contract Child is A, B, C

继承使用方法

1

使用 is 关键字

合约通过 is 关键字实现继承

2

函数重写

使用 override 关键字重写父合约函数

3

构造函数继承

子合约构造函数中调用父合约构造函数

4

super 关键字

调用父合约函数，可调用整个继承链

继承代码示例

contract OwnablePausableStorage is Ownable, Pausable, StorageBase { uint256 private _value; // 构造函数，调用父合约构造函数 constructor(uint256 initialValue) Ownable() { storedValue = initialValue; } // 重写 store 函数，添加暂停检查 function store(uint256 value) public override whenNotPaused { super.store(value); // 调用父合约函数 } // 重写 pause 函数，添加所有者限制 function pause() public override onlyOwner { super.pause(); } }

抽象合约

抽象合约可以包含部分实现和状态变量，子合约必须实现所有抽象方法。

// 抽象合约定义 abstract contract AbstractToken { mapping(address => uint256) public balances; // 抽象函数：没有实现，需要子类实现 function transfer(address to, uint256 amount) public virtual returns (bool); // 具体函数：有默认实现 function balanceOf(address account) public view returns (uint256) { return balances[account]; } } // 实现抽象合约 contract ConcreteToken is AbstractToken { function transfer(address to, uint256 amount) public override returns (bool) { require(balances[msg.sender] >= amount, "Insufficient balance"); balances[msg.sender] -= amount; balances[to] += amount; return true; } }

接口、继承、抽象合约对比

特性 接口 Interface 继承 Inheritance 抽象合约 Abstract
关键字 interface is abstract contract
状态变量 ❌ 不能有 ✅ 可以有 ✅ 可以有
函数实现 ❌ 不能有 ✅ 可以有 ✅ 部分实现
目的 跨合约调用 代码复用 通用逻辑 + 规范
第三部分

库 Library

封装通用工具函数，Gas 优化

库概念介绍

库（Library）是 Solidity 中用于封装通用功能的机制。库函数通过 delegatecall 方式调用，可以实现代码复用和 Gas 优化。

库的特点：

  • 不能有状态变量
  • 不能接收 Ether
  • 函数只能是 internal 或 public
  • 通过 delegatecall 调用，更省 Gas

库使用方法

// 1. 定义库 library SafeMath { function add(uint256 a, uint256 b) internal pure returns (uint256) { uint256 c = a + b; require(c >= a, "SafeMath: addition overflow"); return c; } function sub(uint256 a, uint256 b) internal pure returns (uint256) { require(b <= a, "SafeMath: subtraction overflow"); return a - b; } } // 2. 使用 using ... for 绑定到类型 contract MathContract { using SafeMath for uint256; function add(uint256 a, uint256 b) public pure returns (uint256) { return a.add(b); // 点操作调用 } }

库核心要点

  • 库 = 操作封装 — 通过 library 关键字定义
  • using 库名 for 类型 — 将库绑定到指定类型
  • 点操作调用 — 绑定后通过 . 调用库方法
  • 无状态 — 库不能存储状态，仅包含纯逻辑函数
  • Gas 优化 — 库函数通过 delegatecall 调用更省 Gas
第四部分

Event 事件

记录合约状态变化，前端监听和日志审计

Event 概念介绍

Event（事件）是 Solidity 中用于记录合约状态变化的机制。事件数据存储在交易日志中，不占用存储空间，Gas 成本低。

事件的重要作用：

  • 前端监听 — 实时更新 UI
  • 日志审计 — 追踪历史记录
  • Gas 成本低 — 存储在日志中，不占用存储
  • Etherscan 可查看 — 区块链浏览器显示事件日志

Event 使用方法

contract EventDemo { // 1. 定义事件 - indexed 参数可用于过滤（最多 3 个） event Transfer( address indexed from, address indexed to, uint256 amount ); function transfer(address to, uint256 amount) public { balances[msg.sender] -= amount; balances[to] += amount; // 2. 使用 emit 触发事件 emit Transfer(msg.sender, to, amount); } } // 匿名事件 - 更省 Gas，但无法通过事件签名过滤 event AnonymousEvent( address indexed user, uint256 data ) anonymous;

indexed 参数详解

indexed 参数

  • 最多 3 个 indexed 参数
  • 存储在日志的 topics 中
  • 可以直接过滤查询
  • 适合地址、ID 等查询字段

非 indexed 参数

  • 数量不限
  • 存储在日志的 data 中
  • 不能直接过滤
  • 适合金额、详情等数据

Event 最佳实践

1

记录重要状态变化

如转账、授权、所有权转移等关键操作

2

合理使用 indexed

最多 3 个，用于查询过滤的字段

3

匿名事件省 Gas

对于频繁触发的事件使用 anonymous

4

便于前端监听

设计好事件结构便于前端实时更新

综合示例

综合使用接口、继承、库和 Event

contract AdvancedWallet is Ownable, Pausable, IWallet { using SafeMath for uint256; using AddressUtils for address; mapping(address => uint256) private _balances; event Deposit(address indexed user, uint256 amount); event Withdraw(address indexed user, uint256 amount); // 实现 IWallet 接口 function deposit() external payable override whenNotPaused { require(msg.value > 0, "Amount must be greater than 0"); _balances[msg.sender] = _balances[msg.sender].add(msg.value); emit Deposit(msg.sender, msg.value); } function withdraw(uint256 amount) external override whenNotPaused { require(_balances[msg.sender] >= amount, "Insufficient balance"); _balances[msg.sender] = _balances[msg.sender].sub(amount); payable(msg.sender).transfer(amount); emit Withdraw(msg.sender, amount); } }

最佳实践总结

接口

  • 命名以 "I" 开头
  • 只定义必要函数
  • 通过接口解耦合约

继承

  • 复用通用逻辑
  • 注意多继承顺序
  • 使用 override 标识

库

  • 封装通用函数
  • 使用 using...for
  • 优化 Gas 成本

Event

  • 记录关键操作
  • 合理使用 indexed
  • 便于前端监听
