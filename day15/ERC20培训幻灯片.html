什么是 ERC20 代币

ERC20 = 一套"代币标准"

ERC20 是 Ethereum Request for Comments 第20号提案，定义了以太坊上代币的技术标准。

🏦 以太坊

去中心化的"银行系统"

💳 ERC20 代币

银行发行的"存折"

📋 ERC20 标准

存折的"使用规则"

2 / 20

为什么需要统一标准？

❌ 没有标准之前

比特币: send(to, amount) 以太坊: transfer(to, amount) 莱特币: sendMoney(to, amount) 每种代币接口不同，钱包和交易所需要单独适配

✅ 有了 ERC20 之后

所有代币都用: transfer(to, amount) ✅ 钱包只需写一次代码 ✅ 交易所快速上线新代币 ✅ DeFi 协议与任何代币交互
3 / 20

ERC20 标准接口

ERC20 定义了 6 个函数 + 2 个事件

📌 必须的函数

函数 作用
totalSupply代币总供应量
balanceOf(地址)查询余额
transfer(地址, 数量)转账
approve(地址, 数量)授权
transferFrom(从, 到, 数量)代理转账
allowance(所有者, 使用者)查询授权额度

📌 必须的事件

  • Transfer - 转账时触发
  • Approval - 授权时触发
4 / 20

核心机制：approve + transferFrom

用户 A

→

approve(B, 1000)

授权 1000 代币

→

transferFrom(A)

从 A 的账户转出代币

💡 为什么需要授权转账？

在 DeFi 场景中（如交易所、借贷协议）：

  1. 用户授权合约可以提取自己的代币
  2. 合约才能自动执行交易（如 swap、流动性挖矿）

测试的规律

✅ 正向测试

正常情况应该成功

转账成功、授权成功

❌ 负向测试

异常情况应该失败

余额不足、转账给零地址

📝 代码示例

// 正向测试 it("应该允许转账代币", async function () { await contract.transfer(addr1.address, 100); expect(await contract.balanceOf(addr1.address)).to.equal(100); }); // 负向测试 it("应该拒绝余额不足的转账", async function () { await expect( contract.transfer(addr1.address, 99999999) ).to.be.revertedWith("ERC20: insufficient balance"); });

ERC20 核心状态变量

// ERC20 标准状态变量 string public name; // 代币名称 string public symbol; // 代币符号 uint8 public decimals; // 小数位数（通常 18） uint256 public totalSupply; // 总供应量 // 地址 → 余额的映射 mapping(address => uint256) public balanceOf; // 授权映射：owner → spender → 授权金额 mapping(address => mapping(address => uint256)) public allowance;

📌 核心函数

函数 作用
transfer()用户自己转账
approve()授权他人
transferFrom()被授权者转账
increaseAllowance()增加授权（安全）
decreaseAllowance()减少授权

使用 OpenZeppelin 实现

📦 引入依赖

import "@openzeppelin/contracts/token/ERC20/ERC20.sol"; import "@openzeppelin/contracts/access/Ownable.sol";

🔗 继承关系

┌─────────────────┐ │ Ownable │ ← 权限管理 └────────┬────────┘ │ ┌────────▼────────┐ │ ERC20 │ ← 代币标准 └────────┬────────┘ │ ┌────────▼────────┐ │ MyERC20Token │ ← 你的代币 └─────────────────┘

✨ 自定义功能

  • mint() - 铸造新代币（仅 owner）
  • burn() - 销毁自己的代币
  • finishMinting() - 结束铸造
16 / 20

_mint() 和 _burn() 的来源

这两个方法定义在 OpenZeppelin 的 ERC20.sol 中

通过 import 和 继承 引入

// OpenZeppelin ERC20.sol 中的实现（简化） function _mint(address account, uint256 amount) internal { require(account != address(0), "mint to zero address"); _totalSupply += amount; _balances[account] += amount; emit Transfer(address(0), account, amount); } // 公开接口 mint() 内部调用 _mint() function mint(address to, uint256 amount) public onlyOwner { _mint(to, amount); // 调用内部实现 }

mint()

public 公开接口

_mint()

internal 内部实现
