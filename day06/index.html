1. 数组（Array）- 概念

数组是 Solidity 中用于存储相同类型元素的集合。

📐 固定长度数组

长度在编译时确定，不可改变

📐 动态数组

长度在运行时确定，可动态增长

声明方式

// 固定长度数组
uint[5] public fixedArray;        // 5个 uint256
address[10] public addressArray; // 10个地址

// 动态数组
uint[] public dynamicArray;       // uint256 动态数组
address[] public addressList;    // 地址动态数组
string[] public names;           // 字符串动态数组

1. 数组 - 常用操作

push() - 添加元素

function pushToArray(uint256 _value) public {
    dynamicArray.push(_value);
}

function pushAddress(address _addr) public {
    addressList.push(_addr);
}

pop() - 移除元素

function popFromArray() public {
    require(dynamicArray.length > 0);
    dynamicArray.pop();
}

length - 获取长度

function getLength() public view returns (uint256) {
    return dynamicArray.length;
}

访问元素

function getElement(uint256 _index) public view returns (uint256) {
    require(_index < dynamicArray.length);
    return dynamicArray[_index];
}

delete - 删除元素

function deleteElement(uint256 _index) public {
    delete dynamicArray[_index]; // 设为默认值
}

1. 数组 - 完整示例

contract ArrayDemo {
    uint[] public numbers;

    // 添加元素
    function addNumber(uint256 _value) public {
        numbers.push(_value);
    }

    // 求和
    function sumArray() public view returns (uint256) {
        uint256 sum = 0;
        for (uint256 i = 0; i < numbers.length; i++) {
            sum += numbers[i];
        }
        return sum;
    }

    // 查找最大值
    function findMax() public view returns (uint256) {
        require(numbers.length > 0);
        uint256 max = numbers[0];
        for (uint256 i = 1; i < numbers.length; i++) {
            if (numbers[i] > max) {
                max = numbers[i];
            }
        }
        return max;
    }
}

⚠️ 注意事项

访问数组元素前必须检查索引边界，避免 out-of-bounds 错误

2. Storage vs Memory vs Calldata

理解三种数据位置是编写高效合约的关键！

💾 Storage

永久存储在区块链上

合约状态变量默认使用

Gas 成本：最高

📝 Memory

临时存储

函数执行期间存在

Gas 成本：中等

📡 Calldata

只读区域

来自交易调用数据

Gas 成本：最低

💡 记忆要点

Storage = 持久化（状态变量） | Memory = 临时（函数内） | Calldata = 只读（外部函数参数）

2. 数据位置 - 使用方法

Storage 使用

contract StorageDemo {
    uint[] public storageArray;

    function addToArray(uint256 _value) public {
        // 直接修改 storage
        storageArray.push(_value);
    }

    // 使用 storage 引用
    function modifyViaRef(uint256 _idx, uint256 _val) public {
        uint256[] storage ref = storageArray;
        ref[_idx] = _val; // 影响原始数据
    }
}

Memory 使用

contract MemoryDemo {
    function processArray(uint256[] memory _arr)
        public pure returns (uint256[] memory) {
        // _arr 是 memory 副本
        for (uint i = 0; i < _arr.length; i++) {
            _arr[i] = _arr[i] * 2;
        }
        return _arr; // 返回修改后的副本
    }

    function createArray(uint256 _size) public pure {
        uint256[] memory arr = new uint256[](_size);
    }
}

📡 Calldata - 只能用于 external 函数

function sumData(uint256[] calldata _data) external pure returns (uint256) {
    // _data 是只读的，无法修改
    // 节省 Gas，不需要复制
}

2. Calldata 补充说明

🎯 关键点：Calldata 与 external 函数

  • calldata 只能用于 external 函数
  • 是 external 函数的默认数据位置，可以省略
  • 只读，无法在函数内修改

函数类型与默认数据位置

函数类型 引用类型参数默认数据位置
external calldata（可省略）
internal / private memory（可省略）

⚠️ 注意

internal 函数不能使用 calldata，因为内部函数在合约内部调用，无法访问 calldata 区域

3. 映射（Mapping）- 概念

映射是键值对数据结构，类似哈希表

mapping(keyType => valueType) public mappingName;

声明方式

// 基本映射
mapping(address => uint256) public balances;      // 地址 -> 余额
mapping(uint256 => address) public idToAddress;  // ID -> 地址
mapping(address => bool) public isMember;        // 地址 -> 是否成员

// 嵌套映射
mapping(address => mapping(address => uint256)) public allowances;
// owner -> spender -> 授权额度

⚠️ 限制

  • keyType 只能是值类型（address, uint256, bytes32, bool）
  • 映射无法遍历，需要配合数组
  • 访问不存在的键返回默认值（0 / false / address(0)）

3. 映射 - 操作与遍历

基本操作

function setBalance(address _addr, uint256 _amount) public {
    balances[_addr] = _amount;
}

function getBalance(address _addr) public view returns (uint256) {
    return balances[_addr];
}

function increaseBalance(address _addr, uint256 _amount) public {
    balances[_addr] += _amount;
}

嵌套映射

function setAllowance(address _owner, address _spender, uint256 _amount) public {
    allowances[_owner][_spender] = _amount;
}

function getAllowance(address _owner, address _spender) public view returns (uint256) {
    return allowances[_owner][_spender];
}

遍历实现（配合数组）

address[] public allAddresses;
mapping(address => bool) public addressExists;

function addAddress(address _addr) public {
    if (!addressExists[_addr]) {
        allAddresses.push(_addr);
        addressExists[_addr] = true;
    }
}

function getAllAddresses() public view returns (address[] memory) {
    return allAddresses;
}

4. 结构体（Struct）

结构体是自定义类型，可组合多个相关变量

声明与赋值语法

struct User {
    uint256 id;
    address addr;
    string name;
}

// 方式一：命名参数（推荐）
User memory user1 = User({
    id: 1,
    addr: msg.sender,
    name: "Alice"
});

// 方式二：位置参数
User memory user2 = User(2, msg.sender, "Bob");

⚠️ 注意：为什么用 User({}) 而不是直接 {...}？

在 Solidity 中，结构体类型就像一个"构造函数"，需要通过 Type({...}) 来创建实例。() 是必需的语法。

Storage vs Memory 引用

User[] public users;

function updateUserName(uint256 _index, string memory _name) public {
    User storage u = users[_index];  // storage 引用，修改影响原始数据
    u.name = _name;
}

function processUser(User memory _user) public pure {
    _user.id = _user.id * 2;  // memory 副本，不影响原始数据
}

5. 最佳实践

数组

  • 固定大小已知时，用固定长度数组省 Gas
  • 访问元素前必须边界检查
  • 避免在循环中删除元素
  • 大数组返回会消耗大量 Gas

Storage/Memory

  • external 函数参数用 calldata 省 Gas
  • 内部修改状态变量时使用 storage 引用
  • 引用类型参数必须指定数据位置

映射

  • 映射无法遍历，需配合数组
  • 记录所有键以便遍历
  • 访问不存在的键返回默认值
  • 键类型只能是值类型

Gas 优化

  • calldata > memory > storage（Gas 成本）
  • 批量操作减少 SSTORE
  • 避免动态大小的 memory 数组

6. 注意事项汇总

⚠️ 数组边界

访问 arr[i] 前必须检查 i < arr.length

⚠️ 数据位置

引用类型作为函数参数必须指定 memory/calldata/storage

⚠️ 映射遍历

映射没有 length，无法直接遍历，必须配合数组

⚠️ Calldata 限制

只能用于 external 函数，是只读的

⚠️ 存储修改

storage 引用修改会影响原始数据，memory 不会

⚠️ 结构体语法

创建结构体实例需要 Type({...}) 语法
