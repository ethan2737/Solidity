🔤 布尔类型 (bool)

基本声明

bool public isActive = true;
bool private isLocked = false;
bool internal isPaused = false;

关键点

  • ✓只有 true 和 false 两个值
  • ✓默认值为 false
  • ✗没有 JavaScript 的 truthy/falsy

对比：Solidity vs JavaScript

❌ JavaScript (动态类型)

// 这些在 Solidity 中都不行！
let a = true;
let b = 1;      // truthy
let c = "hello"; // truthy
let d = 0;      // falsy

✅ Solidity (静态类型)

bool a = true;
bool b = 1;     // ❌ 编译错误！
bool c = 0;     // ❌ 编译错误！
// 只能是 true 或 false

🔢 整型类型

声明示例

uint8 public smallNumber = 100;     // 0 ~ 255
uint256 public largeNumber = 1000000; // 超大范围
uint public defaultUint = 42;        // uint = uint256

💡 提示：uint 是 uint256 的简写

类型范围

uint8 0 ~ 255
uint16 0 ~ 65,535
uint32 0 ~ 4,294,967,295
uint256 0 ~ 2²⁵⁶-1

声明示例

int8 public smallInt = -50;       // -128 ~ 127
int256 public largeInt = -1000000; // 大范围有符号
int public defaultInt = -42;      // int = int256

类型范围

int8 -128 ~ 127
int16 -32,768 ~ 32,767
int256 -2²⁵⁵ ~ 2²⁵⁵-1

💡 注意：int 可以表示负数

使用 type(X).max / type(X).min 获取边界值

int256 public int256Max = type(int256).max;
int256 public int256Min = type(int256).min;
uint256 public uint256Max = type(uint256).max;
type(int256).max

= 2²⁵⁵ - 1

type(int256).min

= -2²⁵⁵

type(uint256).max

= 2²⁵⁶ - 1

👁️ 可见性修饰符

变量可见性（3种）

🌐 public

全部可见

  • ✓ 自动生成 getter
  • ✓ 合约内外可访问
  • ✓ 外部可直接读取
🔒 internal

本合约 + 继承合约

  • ✓ 默认可见性
  • ✓ 子合约可访问
  • ✗ 外部不可访问
🔐 private

仅本合约

  • ✓ 仅当前合约可见
  • ✗ 子合约不可访问
  • ✗ 外部不可访问

函数可见性（4种）

public

内外都可调用

external

仅外部可调用（更省 gas）

internal

内部 + 继承调用

private

仅内部调用

uint256 public publicVar = 100;     // 外部可访问，有 getter
uint256 internal internalVar = 200; // 本合约+子合约可用
uint256 private privateVar = 300;   // 仅本合约能用

function externalFunc() external {}  // 只能外部调，更省 gas
function publicFunc() public {}      // 内外都能调

🔒 常量与不可变变量

📌

constant

编译时确定，写死在代码中
不占用存储槽位（gas 几乎为 0）
声明时必须赋值 
uint256 public constant MAX_VALUE = 1000000;
🛡️

immutable

构造函数中设置，部署时确定
不占用存储槽位（较省 gas）
部署后不可更改 
uint256 public immutable INITIAL_VALUE;

constructor(uint256 _initialValue) {
    INITIAL_VALUE = _initialValue;
}
特性 constant immutable
赋值时机 编译时 构造函数
存储位置 嵌入字节码 嵌入字节码
Gas 成本 几乎为 0 较低
值来源 代码硬编码 部署参数
典型用途 固定配置 部署时配置

✅ 条件检查与构造函数

✅ require 条件检查

语法：

require(条件, "错误消息");

条件为真 → 继续执行 ✅

条件为假 → 回滚交易 ❌

💡 口诀：条件为真继续走，条件为假就回头！

实际应用

function setSmallNumber(uint8 _value) public {
    // 检查：值必须 <= 255
    require(
        _value <= 255,
        "Value too large for uint8"
    );

    // 检查通过，继续执行
    smallNumber = _value;
}
✓ 传入 100 → 通过 → 赋值成功
✗ 传入 300 → 失败 → 报错回滚

🏗️ 构造函数 (constructor)

合约部署时自动执行一次，用于初始化：

  • • 设置 immutable 变量
  • • 初始化合约状态
  • • 配置权限地址 
constructor(uint256 _initialValue) {
    INITIAL_VALUE = _initialValue;
    owner = msg.sender;
}
