1.1 msg.sender 与 msg.value

概念 msg.sender：当前调用函数的直接发送者地址

概念 msg.value：随消息发送的以太币数量（wei）

// 存款功能示例
function deposit() public payable {
    // msg.sender: 调用者地址
    // msg.value: 发送的 ETH 数量
    balances[msg.sender] += msg.value;
}

⚠️ 注意：msg.sender 是最常用的权限验证变量

msg.value 仅在 payable 函数中可用

1.2 tx.origin

tx.origin：整个交易链的原始发起者地址

无论经过多少个合约转发，始终是最初发起交易的外部账户

function getOrigin() public view returns (address) {
    return tx.origin;
}

⚠️ 重要警告：不要使用 tx.origin 进行权限检查！

tx.origin 攻击原理

攻击流程：

  1. 攻击者构造恶意合约
  2. 诱骗 owner 调用恶意合约
  3. 恶意合约调用目标合约的 withdraw
  4. 目标合约检查 tx.origin == owner 通过！
  5. 攻击成功，资金被盗

✅ 解决方案：始终使用 msg.sender 进行权限检查

安全对比

❌ 危险写法

function withdraw() public {
    require(tx.origin == owner,
        "Not owner");
    // ...
}

✅ 安全写法

function withdraw() public {
    require(msg.sender == owner,
        "Not owner");
    // ...
}

2.1 require

用于输入验证和条件检查

失败时回滚所有状态变化

function setValue(uint256 _value) public {
    require(_value > 0, "Value must be > 0");
    require(_value <= 1000, "Value too large");
    require(msg.sender == owner, "Not owner");

    value = _value;
}

使用场景：

  • 输入参数验证
  • 权限检查
  • 状态前置条件

2.2 revert

无条件回滚当前执行

适合复杂条件判断

function setValue(uint256 _value) public {
    if (_value == 0) {
        revert("Value cannot be zero");
    }

    if (_value > 1000) {
        revert("Value exceeds maximum");
    }

    value = _value;
}

与 require 的区别：require 适合单条件，revert 适合复杂多层条件

2.3 assert

用于内部错误检查

断言合约的不变量，失败表示合约存在 bug

function deposit(uint256 amount) public {
    require(amount > 0, "Amount must be > 0");

    uint256 oldBalance = balances[msg.sender];
    balances[msg.sender] += amount;

    // assert 检查不变量
    assert(balances[msg.sender] > oldBalance);
}
assert require
用于内部错误检查 用于输入验证
失败 = 合约 bug 失败 = 输入无效

2.4 try/catch (Solidity 0.6+)

用于处理外部合约调用错误

不会中断执行，可以捕获不同类型的错误

try target.setValue(_value) {
    // 调用成功
    value = _value;
} catch Error(string memory reason) {
    // 捕获 require/revert 错误
} catch (bytes memory) {
    // 捕获低级错误
}

catch 类型：

  • Error(string) - 字符串错误
  • Panic(uint) - assert 失败
  • bytes - 低级错误

2.5 自定义错误 (Solidity 0.8.4+)

使用 error 关键字定义

比字符串错误更节省 Gas，更清晰的错误类型

// 定义自定义错误
error InsufficientBalance(uint256 requested, uint256 available);
error Unauthorized(address caller);

// 使用自定义错误
function withdraw(uint256 amount) public {
    if (balances[msg.sender] < amount) {
        revert InsufficientBalance(amount, balances[msg.sender]);
    }
    // ...
}

💰 Gas 对比：字符串错误 ~100+ Gas/字符，自定义错误 4-8 Gas

3. 安全最佳实践

❌ 禁止

  • 使用 tx.origin 权限检查
  • 依赖 block.timestamp 精确判断
  • 使用长字符串错误消息

✅ 推荐

  • 使用 msg.sender 权限检查
  • 使用时间窗口而非精确时间
  • 生产环境使用自定义错误

错误处理最佳实践：

  1. 优先使用 require 进行输入验证
  2. 复杂条件使用 revert
  3. 内部检查使用 assert
  4. 外部调用使用 try/catch

4. 最小可行案例：银行合约

// SPDX-License-Identifier: MIT
 solidity ^0 class="keyword">.8.24;

contract BankDemo {
    address public owner;
    mapping(address => uint256) public balances;

    // 自定义错误
    error InsufficientBalance(uint256 requested, uint256 available);

    constructor() {
        owner = msg.sender;
    }

    // 存款
    function deposit() public payable {
        require(msg.value > 0, "Must send ETH");
        balances[msg.sender] += msg.value;
    }

    // 提款 - 安全实现
    function withdraw(uint256 amount) public {
        require(balances[msg.sender] >= amount, "Insufficient balance");
        balances[msg.sender] -= amount;
        payable(msg.sender).transfer(amount);
    }
}

4. tx.origin 攻击演示

⚠️ VulnerableWallet

function withdraw(uint256 amount) {
    require(tx.origin == owner,
        "Only owner");
    // 漏洞！
}

✅ SafeWallet

function withdraw(uint256 amount) {
    require(balances[msg.sender]
        >= amount, "Insufficient");
    // 安全！
}

攻击者合约：AttackerContract 诱骗 owner 调用，然后调用 VulnerableWallet.withdraw()，由于 tx.origin == owner 检查通过，资金被盗。

5. 使用注意事项

全局变量注意事项

  • msg.sender 可能为 0 地址：首次调用时需验证
  • msg.value 是 wei：大额计算注意单位换算
  • block.timestamp 可被操纵：仅用于大致判断
  • gasleft() 有最低消耗：剩余 Gas 可能不足

错误处理注意事项

  • require 消耗剩余 Gas（旧版本）
  • revert 可以嵌套但不要过度复杂
  • assert 失败会消耗所有 Gas
  • try/catch 只捕获外部调用错误

📝 总结

核心要点

  • ✅ 永远不要使用 tx.origin 进行权限检查
  • ✅ 优先使用自定义错误节省 Gas
  • ✅ try/catch 用于处理外部调用错误
  • ✅ 正确使用错误处理是安全合约的基础
