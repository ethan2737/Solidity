## 第一部分：全局变量与函数 (Day13)

### 1.1 msg.sender 与 msg.value

**概念解释**：
- `msg.sender`：当前调用函数的直接发送者地址（可以是外部账户 EOA 或合约地址）
- `msg.value`：随消息发送的以太币数量，单位为 wei

**使用方法**：
```solidity
function deposit() public payable {
    // msg.sender: 获取调用者的地址
    // msg.value: 获取发送的 ETH 数量（wei）
    balances[msg.sender] += msg.value;
}
```

**注意事项**：
- `msg.sender` 是最常用的权限验证变量
- 在合约调用合约的场景下，`msg.sender` 会改变
- `msg.value` 仅在 `payable` 函数中可用

---

### 1.2 tx.origin

**概念解释**：
- `tx.origin`：整个交易链的原始发起者地址
- 无论经过多少个合约转发，`tx.origin` 始终是最初发起交易的外部账户

**使用方法**：
```solidity
function getOrigin() public view returns (address) {
    return tx.origin;
}
```

**重要警告 - tx.origin 攻击**：
```solidity
// ⚠️ 危险：不安全的实现
function withdraw(uint256 amount) public {
    require(tx.origin == owner, "Only owner");  // 存在安全漏洞
    // ...
}

// ✅ 安全：使用 msg.sender
function withdraw(uint256 amount) public {
    require(msg.sender == owner, "Only owner");  // 安全
    // ...
}
```

**攻击原理**：
1. 攻击者构造恶意合约
2. 诱骗 owner 调用恶意合约
3. 恶意合约调用目标合约的 withdraw
4. 目标合约检查 `tx.origin == owner` 通过
5. 攻击成功

### 1.3 区块相关全局变量

| 变量 | 说明 | 注意事项 |
|------|------|----------|
| `block.timestamp` | 当前区块时间戳（Unix秒） | 矿工可操纵 ±15 秒 |
| `block.number` | 当前区块号 | 相对可靠 |
| `block.coinbase` | 当前区块矿工地址 | 可被操纵 |
| `block.difficulty` | 当前区块难度 | PoS 中含义不同 |
| `block.gaslimit` | 当前区块 Gas 上限 | 约 3000 万 |
| `block.chainid` | 当前链 ID | 区分不同链 |

**使用方法**：
```solidity
function getBlockInfo() public view returns (
    uint256 timestamp,
    uint256 number,
    address coinbase
) {
    return (block.timestamp, block.number, block.coinbase);
}
```

**注意事项**：
- 不要用 `block.timestamp` 做精确时间判断
- 只能用于大致的时间范围判断
- `block.number` 可用于估算时间（每块约 12 秒）

### 1.5 加密与编码函数

#### 1.5.1 abi 编码函数

| 函数 | 说明 |
|------|------|
| `abi.encode(...)` | 标准 ABI 编码（固定长度，有填充） |
| `abi.encodePacked(...)` | 紧密编码（可变长度，无填充） |
| `abi.encodeWithSelector(...)` | 带函数选择器编码 |
| `abi.encodeWithSignature(...)` | 带函数签名编码 |

**使用示例**：
```solidity
function encodeDemo(uint256 a, uint256 b) public pure returns (bytes memory) {
    // 标准 ABI 编码
    bytes memory encoded = abi.encode(a, b);

    // 紧密编码（用于哈希）
    bytes memory packed = abi.encodePacked(a, b);

    return encoded;
}
```

**注意事项**：
- `abi.encodePacked` 存在哈希碰撞风险
- 不同参数可能产生相同的编码结果
- 使用时注意参数类型和顺序

#### 1.5.2 keccak256 哈希函数

**使用示例**：
```solidity
function hashDemo(string memory message) public pure returns (bytes32) {
    // 将字符串编码后哈希
    return keccak256(abi.encodePacked(message));
}

function hashMultiple(uint256 a, address b) public pure returns (bytes32) {
    // 多参数哈希
    return keccak256(abi.encodePacked(a, b));
}
```

**应用场景**：
- 数据完整性验证
- 生成唯一标识
- 签名验证

#### 1.5.3 ecrecover 签名验证

**使用示例**：
```solidity
function recoverSigner(
    bytes32 messageHash,
    uint8 v,
    bytes32 r,
    bytes32 s
) public pure returns (address) {
    return ecrecover(messageHash, v, r, s);
}
```

**注意事项**：
- 需要防止重放攻击（使用 nonce）
- 消息应包含合约地址和 nonce

## 第二部分：错误处理 (Day14)

### 2.1 require

**概念解释**：
- 用于输入验证和条件检查
- 失败时回滚所有状态变化
- 可附带错误消息

**使用方法**：
```solidity
function setValue(uint256 _value) public {
    require(_value > 0, "Value must be greater than 0");
    require(_value <= 1000, "Value must be less than or equal to 1000");
    require(msg.sender == owner, "Only owner");

    value = _value;
}
```

**使用场景**：
- 输入参数验证
- 权限检查
- 状态前置条件检查
- 余额检查

---

### 2.2 revert

**概念解释**：
- 无条件回滚当前执行
- 可以带错误消息或自定义错误
- 更适合复杂条件判断

**使用方法**：
```solidity
function setValue(uint256 _value) public {
    if (_value == 0) {
        revert("Value cannot be zero");
    }

    if (_value > 1000) {
        revert("Value exceeds maximum");
    }

    value = _value;
}
```

**与 require 的区别**：
- `require` 适合单条件判断
- `revert` 适合复杂多层条件判断
- 两者都会回滚状态

---

### 2.3 assert

**概念解释**：
- 用于内部错误检查
- 断言合约的不变量
- 失败表示合约存在 bug

**使用方法**：
```solidity
function deposit(uint256 amount) public {
    require(amount > 0, "Amount must be greater than 0");

    uint256 oldBalance = balances[msg.sender];
    balances[msg.sender] += amount;

    // assert 检查不变量：余额应该增加
    assert(balances[msg.sender] > oldBalance);
}
```

**与 require 的区别**：

| assert | require |
|--------|---------|
| 用于内部错误检查 | 用于输入验证 |
| 失败 = 合约 bug | 失败 = 输入无效 |
| 不应发生 | 可能发生 |

---

### 2.4 try/catch (Solidity 0.6+)

**概念解释**：
- 用于处理外部合约调用错误
- 不会中断执行
- 可以捕获不同类型的错误

**使用方法**：
```solidity
function callExternal(address _contract, uint256 _value) public {
    TargetContract target = TargetContract(_contract);

    try target.setValue(_value) {
        // 调用成功
        value = _value;
    } catch Error(string memory reason) {
        // 捕获 require/revert 错误
        revert(string(abi.encodePacked("Call failed: ", reason)));
    } catch (bytes memory) {
        // 捕获低级错误（assert 失败）
        revert("Low-level error");
    }
}
```

**catch 分支类型**：
- `catch Error(string)`：捕获 require/revert 字符串错误
- `catch Panic(uint)`：捕获 assert 失败（Solidity 0.8+）
- `catch (bytes)`：捕获低级错误
- `catch`：捕获所有错误

---

### 2.4.1 try/catch 深入理解

#### 为什么要先声明外部合约接口？

这是 Solidity 的固定语法，因为：
1. **类型转换**：`address` 类型本身不能直接调用函数，必须转换为合约类型
2. **类型安全**：编译器需要知道目标合约有哪些函数

```solidity
// 写法1：直接使用合约类型（需要目标合约源码）
SolidityAPIDemo target = SolidityAPIDemo(_target);

// 写法2：使用接口（Interface，更灵活）
interface IExample {
    function setValue(uint256) external;
}
IExample target = IExample(_target);
```

#### _target 是什么？

`_target` 是**目标合约的地址**（即你要调用哪个合约）。

```solidity
// 调用示例
callExternalWithTryCatch(0x1234...def, 100);
//                ↑ 目标合约地址  ↑ 要传入的值
```

#### 目标合约指的是本合约还是其他合约？

**可以是本合约，也可以是其他合约**。

```solidity
// 调用自己的合约
SolidityAPIDemo target = SolidityAPIDemo(address(this));
target.setValueWithRequire(100);

// 调用其他合约（需要知道地址和函数签名）
SolidityAPIDemo target = SolidityAPIDemo(0xABC...);
target.setValueWithRequire(100);
```

#### try/catch 是固定写法吗？

**是固定模式**，标准写法如下：

```solidity
try 目标合约.函数(参数) {
    // 成功时执行
} catch Error(string memory reason) {
    // 捕获 require/revert 字符串错误
} catch Panic(uint256) {
    // 捕获 assert 失败 (Solidity 0.8+)
} catch (bytes memory lowLevelData) {
    // 捕获其他所有错误
} catch {
    // 捕获所有错误（不关心内容）
}
```

#### 实际使用场景

```solidity
// 场景：批量调用多个合约，部分失败不影响其他
function batchCall(address[] calldata targets, uint256[] calldata values) public {
    for (uint i = 0; i < targets.length; i++) {
        try SolidityAPIDemo(targets[i]).setValueWithRequire(values[i]) {
            // 成功
        } catch {
            // 失败，继续处理下一个
        }
    }
}
```

**简单理解**：try/catch 就像 JavaScript 的 try/catch，用于捕获外部调用（跨合约调用）可能失败的错误，避免整个交易 revert。

### 2.5 自定义错误 (Solidity 0.8.4+)

**概念解释**：
- 使用 `error` 关键字定义
- 比字符串错误更节省 Gas
- 更清晰的错误类型

**使用方法**：
```solidity
// 定义自定义错误
error InsufficientBalance(uint256 requested, uint256 available);
error Unauthorized(address caller);
error TransferFailed();

// 使用自定义错误
function withdraw(uint256 amount) public {
    if (balances[msg.sender] < amount) {
        revert InsufficientBalance(amount, balances[msg.sender]);
    }

    if (msg.sender != owner) {
        revert Unauthorized(msg.sender);
    }

    // ...
}
```

**Gas 对比**：
- 字符串错误：约 100+ Gas/字符
- 自定义错误：固定 4-8 Gas

## 第三部分：安全最佳实践

### 3.1 tx.origin 安全指南

1. **永远不要使用 tx.origin 进行权限检查**
2. **始终使用 msg.sender 进行授权**
3. **在必要时使用 tx.origin 进行信息记录**

```solidity
// ✅ 正确：使用 msg.sender
function withdraw() public {
    require(msg.sender == owner, "Not authorized");
    // ...
}

// ⚠️ 错误：使用 tx.origin
function withdraw() public {
    require(tx.origin == owner, "Not authorized");
    // ...
}
```

### 3.2 错误处理最佳实践

1. **优先使用 require 进行输入验证**
2. **复杂条件使用 revert**
3. **内部一致性检查使用 assert**
4. **外部调用使用 try/catch**
5. **生产环境使用自定义错误**

### 3.3 时间相关注意事项

1. **不要依赖 block.timestamp 做关键决策**
2. **使用时间窗口而非精确时间**
3. **考虑 15 秒的容差范围**

## 第四部分：最小可行案例

### 案例 1：带权限控制的银行合约

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract BankDemo {
    // 状态变量
    address public owner;
    mapping(address => uint256) public balances;

    // 自定义错误（Gas 优化）
    error InsufficientBalance(uint256 requested, uint256 available);
    error Unauthorized(address caller);
    error ZeroAmountNotAllowed();

    // 事件
    event Deposit(address indexed account, uint256 amount);
    event Withdraw(address indexed account, uint256 amount);

    // 构造函数：部署者成为 owner
    constructor() {
        owner = msg.sender;
    }

    // 存款功能
    function deposit() public payable {
        require(msg.value > 0, ZeroAmountNotAllowed());
        balances[msg.sender] += msg.value;
        emit Deposit(msg.sender, msg.value);
    }

    // 提取功能（安全实现）
    function withdraw(uint256 amount) public {
        require(amount > 0, ZeroAmountNotAllowed());
        require(balances[msg.sender] >= amount, InsufficientBalance({
            requested: amount,
            available: balances[msg.sender]
        }));

        balances[msg.sender] -= amount;
        payable(msg.sender).transfer(amount);
        emit Withdraw(msg.sender, amount);
    }

    // 获取余额
    function getBalance(address account) public view returns (uint256) {
        return balances[account];
    }
}
```

### 案例 2：tx.origin 攻击演示

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

/**
 * 易受攻击的钱包（使用 tx.origin）
 * ⚠️ 仅用于演示，不要在实际项目中使用
 */
contract VulnerableWallet {
    address public owner;
    mapping(address => uint256) public balances;

    constructor() {
        owner = msg.sender;
    }

    function deposit() public payable {
        balances[msg.sender] += msg.value;
    }

    // ⚠️ 危险：使用 tx.origin 进行权限检查
    function withdraw(uint256 amount) public {
        require(tx.origin == owner, "Only owner");  // 漏洞所在
        require(balances[tx.origin] >= amount, "Insufficient balance");

        balances[tx.origin] -= amount;
        payable(tx.origin).transfer(amount);
    }
}

/**
 * 安全的钱包（使用 msg.sender）
 * ✅ 正确实现
 */
contract SafeWallet {
    address public owner;
    mapping(address => uint256) public balances;

    constructor() {
        owner = msg.sender;
    }

    function deposit() public payable {
        balances[msg.sender] += msg.value;
    }

    // ✅ 安全：使用 msg.sender 进行权限检查
    function withdraw(uint256 amount) public {
        require(balances[msg.sender] >= amount, "Insufficient balance");

        balances[msg.sender] -= amount;
        payable(msg.sender).transfer(amount);
    }

    // owner 提取所有资金
    function withdrawAll() public {
        require(msg.sender == owner, "Only owner");
        payable(owner).transfer(address(this).balance);
    }
}

/**
 * 攻击者合约
 * 演示如何利用 VulnerableWallet 的 tx.origin 漏洞
 */
contract AttackerContract {
    VulnerableWallet public vulnerableWallet;

    constructor(address _vulnerableWallet) {
        vulnerableWallet = VulnerableWallet(_vulnerableWallet);
    }

    // 攻击函数：诱骗 owner 调用后提取资金
    function attack(uint256 amount) public {
        vulnerableWallet.withdraw(amount);
    }

    receive() external payable {}
}
```

### 案例 3：外部调用错误处理

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

// 目标合约接口
interface IErrorContract {
    function setValue(uint256 _value) external;
}

contract ErrorHandlingDemo {
    uint256 public value;
    address public owner;

    // 自定义错误
    error CallFailed(string reason);
    error InvalidValue(uint256 value);

    constructor() {
        owner = msg.sender;
    }

    // 使用 try/catch 处理外部调用
    function callWithTryCatch(
        address _contract,
        uint256 _value
    ) public returns (bool success, string memory errorMessage) {
        IErrorContract target = IErrorContract(_contract);

        try target.setValue(_value) {
            value = _value;
            return (true, "");
        } catch Error(string memory reason) {
            return (false, reason);
        } catch (bytes memory) {
            return (false, "Low-level error");
        }
    }

    // 批量调用：部分失败不影响其他调用
    function batchCall(
        address[] calldata contracts,
        uint256[] calldata values
    ) public returns (uint256 successCount) {
        require(contracts.length == values.length, "Length mismatch");

        successCount = 0;
        for (uint256 i = 0; i < contracts.length; i++) {
            try IErrorContract(contracts[i]).setValue(values[i]) {
                successCount++;
            } catch {
                // 忽略错误，继续执行
            }
        }
    }
}
```

## 第五部分：使用注意事项

### 5.1 全局变量注意事项

1. **msg.sender 可能为 0 地址**：首次调用时需验证
2. **msg.value 是 wei**：大额计算注意单位换算
3. **block.timestamp 可被操纵**：仅用于大致判断
4. **gasleft() 有最低消耗**：剩余 Gas 可能不足

### 5.2 错误处理注意事项

1. **require 消耗剩余 Gas**：旧版本行为
2. **revert 可以嵌套**：但不要过度复杂
3. **assert 失败会消耗所有 Gas**
4. **try/catch 只捕获外部调用错误**

### 5.3 Gas 优化建议

1. **使用自定义错误替代字符串**
2. **减少 require 检查次数**
3. **合理使用 revert 而非 require**
4. **避免在错误消息中使用长字符串**

本笔记涵盖了 Solidity 中两个重要的主题：

1. **全局变量与函数**：理解 msg.sender、tx.origin、block.* 等的区别和安全使用

**核心要点**：

- try/catch 用于处理外部调用错误
- 正确使用错误处理是安全合约的基础
