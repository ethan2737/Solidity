1 什么是 EVM

EVM（Ethereum Virtual Machine，以太坊虚拟机）是运行智能合约的虚拟计算机

  • 隔离的执行环境 — 智能合约在"沙箱"中运行
  • Gas 机制 — 每次函数调用消耗 Gas（燃料）
  • 费用由调用者支付 — 激励开发者编写高效代码

2 什么是 Gas

Gas 是执行智能合约操作所需的计算资源单位 — 就像汽车的燃油

  • 每个操作（如存储数据、计算哈希）都有固定的 Gas 成本
  • Gas 价格由市场决定，但每个操作有 最低消耗
  • Gas 退还机制：revert 会退还剩余 Gas，assert 失败不退

第一章：自定义数据类型

在 Solidity 中，我们不仅可以使用基本类型，还可以创建自定义类型

📦 结构体 (Struct)

将多个相关数据组合成一个逻辑单元

🔑 映射 (Mapping)

键值对数据结构，提供 O(1) 的快速查找

1.1 结构体 (Struct) — 定义

将多个不同类型的变量"打包"在一起

Solidity 
struct User {
    address addr;      // 用户地址
    uint256 balance;  // 用户余额
    bool isActive;    // 是否激活
}

类比：结构体就像其他语言的 struct 或 class，用于组织复杂数据

1.1 结构体 — 初始化方式

✅ 命名参数（推荐）

明确指定每个字段的值，字段顺序无关，代码可读性好

⚠️ 位置参数

按照结构体定义字段的顺序依次传入值，容易因顺序错误导致赋值错误

Solidity 
// 方式一：命名参数（推荐）
User memory user1 = User({
    addr: msg.sender,
    balance: 100,
    isActive: true
});

// 方式二：位置参数（不推荐）
User memory user2 = User(msg.sender, 100, true);

1.1 结构体 — 比较与复制

结构体不能直接用 == 比较，需要用 keccak256 哈希后比较

Solidity 
struct Person {
    string name;
    uint256 age;
}

function compare(Person calldata p1, Person calldata p2)
    public pure returns (bool) {
    return keccak256(bytes(p1.name)) == keccak256(bytes(p2.name))
        && p1.age == p2.age;
}

// 结构体赋值默认是值拷贝（深拷贝）
function copy(Person calldata p)
    public pure returns (Person memory) {
    return Person(p.name, p.age);
}

1.2 映射 (Mapping) — 概念

映射 — 键值对数据结构，查找效率极高

  • O(1) 时间复杂度 — 无论数据量多大，查找速度一样快
  • 类比：映射就像字典书后的索引页，直接翻到目标页
  • O(n)：数组遍历就像从头逐页翻书

1.2 映射 — 定义语法

Solidity 
// 映射定义语法：mapping(键类型 => 值类型) 可见性 名称;

// 键类型（KeyType）：只能是值类型
// 值类型（ValueType）：可以是任何类型

mapping(address => uint256) public balances;    // 地址 → 余额
mapping(uint256 => string) public names;        // ID → 名称
mapping(address => bool) public isRegistered;   // 地址 → 是否注册

⚠️ 键类型只能是值类型（address、uint256、bytes32、bool、enum），不能是 mapping、数组、struct

1.2 映射 — 访问行为

访问不存在的键时，返回该类型的默认值

Solidity 
mapping(address => uint256) public balances;

function getBalance(address _addr)
    public view returns (uint256) {
    return balances[_addr];  // 从未设置过返回 ？0
}

问题：无法区分"值是 0"和"键不存在"的情况

1.2 映射 — 存在性检查

配合一个额外的 bool 映射来检查数据是否存在

Solidity 
mapping(address => uint256) public balances;
mapping(address => bool) public hasBalance;

function setBalance(address _addr, uint256 _amount) public {
    balances[_addr] = _amount;
    hasBalance[_addr] = true;  // 标记已设置
}

function getBalance(address _addr)
    public view returns (uint256, bool) {
    return (balances[_addr], hasBalance[_addr]);
}

1.2 映射 — 遍历问题

映射的底层是哈希表，只存储键值对，不存储键的列表

  • ❌ 无法获取"所有键"
  • ❌ 无法知道映射的大小
  • ❌ 无法用 for 循环遍历

解决方案：使用辅助数组同步存储键列表

1.3 嵌套结构与组合

🔄 嵌套映射

mapping(address => mapping(uint256 => uint256))

📦 映射 + 结构体

以地址/ID为键，快速查找复杂的用户信息

Solidity 
struct User {
    address addr;
    uint256 balance;
}

mapping(address => User) public users;
mapping(address => bool) public userExists;

第二章：数据存储位置

智能合约的状态需要持久化存储，但区块链存储昂贵

位置 说明 生命周期 Gas 成本
Storage 永久存储在区块链上 合约存活期间 高
Memory 临时存储在内存中 函数调用期间 低
Calldata 不可修改的临时存储 函数调用期间 最低

数据存储位置 — 使用规则

Solidity 
// 状态变量（默认 storage）
Person public person;

// 函数参数：Memory 或 Calldata
function foo(Person memory p) internal pure {
    p.age = 100;  // 修改的是副本
}

function bar(Person calldata p) internal pure {
    // p.age = 100; // 编译错误：calldata 不可修改
    uint256 a = p.age;  // 可以读取
}

// 返回值：必须是 Memory
function getPerson() public view
    returns (Person memory) {
    return person;
}

理解函数

函数（Function） 是执行特定任务的代码块，是用户与合约交互的唯一方式

Solidity 
// 函数名    参数          返回值
function setValue(uint256 _value) public returns (bool) {
    value = _value;
    return true;
}
  • 传统程序：函数像 API 接口
  • 智能合约：函数是用户与合约交互的唯一方式

第三章：函数可见性

控制"谁能调用这个函数"

可见性 调用范围 特点
public 内部 + 外部 可被任何人调用
external 仅外部 Gas 效率更高
internal 仅内部 当前合约及子类
private 仅当前合约 不能被继承

函数可见性 — 示例

Solidity 
function publicFunc(uint256 _value) public { }
function externalFunc(uint256 _value) external { }
function _internalFunc(uint256 _value) internal { }
function _privateFunc(uint256 _value) private { }

// 内部调用
function callInternal() public {
    _internalFunc(100);
}

// 外部调用（通过 this）
function callExternal() public {
    this.externalFunc(200);
}

最佳实践：如果函数不需要内部调用，优先使用 external

函数调用方式

⚡ 内部调用

直接在当前合约代码中执行，不产生新的调用帧

Gas 消耗少 | 栈深度限制宽松

🌐 外部调用

产生新的 EVM 调用帧，像调用另一个合约

Gas 消耗多 | 栈深度限制严格

⚠️ 外部调用栈深度约 16 层，内部调用约 1024 层

错误处理 — 三种方式

关键字 用途 Gas 退还 典型场景
require 条件验证 是 输入检查、权限检查
revert 主动回滚 是 条件分支回滚
assert 内部检查 否 不应该发生的错误

错误处理 — 示例

Solidity 
// require：输入验证、权限检查（最常用）
function setValueRequire(uint256 _value) public {
    require(_value > 0, "Value must be > 0");
    require(_value <= 1000, "Value must be <= 1000");
    value = _value;
}

// assert：检查内部不变量
function setValueAssert(uint256 _value) public {
    uint256 oldValue = value;
    value = _value;
    assert(value >= oldValue);  // 失败=严重bug
}

自定义错误 (Solidity 0.8.4+)

字符串错误消息占用存储空间，自定义错误只存储错误名称和参数，效率更高

Solidity 
// 定义自定义错误
error InsufficientBalance(uint256 requested, uint256 available);
error Unauthorized(address caller);

// 使用自定义错误
function withdraw(uint256 _amount) public {
    uint256 balance = balances[msg.sender];
    if (balance < _amount) {
        revert InsufficientBalance(_amount, balance);
    }
    balances[msg.sender] -= _amount;
}

综合最佳实践

📋 函数设计

优先使用 external | 优先内部调用 | Internal/Private 用 _ 前缀

⚠️ 错误处理

输入验证用 require | 不变量用 assert | 外部调用用 try-catch

📦 数据结构

结构体用命名参数 | 映射配合存在性检查 | 遍历用辅助数组

💾 存储位置

状态变量默认 Storage | 参数不可修改用 Calldata | 修改结构体用 storage 引用
