01 枚举

什么是枚举？

枚举是一种用户自定义类型，用于表示有限数量的状态值。让代码更易读、更安全。

生活中的例子

  • 星期几：Monday, Tuesday, Wednesday...
  • 订单状态：Pending, Confirmed, Shipped, Delivered
  • 用户角色：Guest, Member, Admin, Owner
01 枚举

枚举的定义语法

// 语法: enum EnumName { Value1, Value2, Value3 } enum State { Created, // 0 - 初始状态 Locked, // 1 - 锁定状态 Inactive // 2 - 非活跃状态 } enum OrderStatus { Pending, // 0 - 待确认 Confirmed, // 1 - 已确认 Shipped, // 2 - 已发货 Delivered, // 3 - 已送达 Cancelled // 4 - 已取消 }
  • 成员值从 0 开始自动编号
  • 可以显式指定值：Created = 0, Locked = 1
01 枚举

枚举的使用

contract EnumDemo { // 定义枚举 enum State { Created, Locked, Inactive } // 使用枚举声明状态变量 State public currentState = State.Created; // 枚举作为映射的值类型 mapping(uint256 => OrderStatus) public orders; }

函数中使用枚举

function lock() public { require( currentState == State.Created, "Can only lock from Created state" ); currentState = State.Locked; }
01 枚举

枚举与整型的转换

枚举 → 整数

State state = State.Locked; uint8 value = uint8(state); // Locked = 1

整数 → 枚举

uint8 value = 1; State state = State(value); // 等同于 State.Locked

⚠️ 注意：需要验证值的有效性！

整数转枚举时要先验证范围，防止无效值。

02 自定义值类型

什么是自定义值类型？

自定义值类型是 Solidity 0.8+ 引入的新特性，基于内置类型创建新的类型。

类型安全

防止意外的类型混用

代码可读性

用 Price 而非 uint256 表示价格

语义明确

代码意图更清晰

02 自定义值类型

自定义值类型的定义

// 语法: type TypeName is UnderlyingType; // 位置: 在合约外部、pragma 之后定义 type UserId is uint128; // 基于 uint128 的用户ID type OrderId is uint256; // 基于 uint256 的订单ID type Price is uint256; // 基于 uint256 的价格 type Percentage is uint8; // 基于 uint8 的百分比

底层类型只能是整数类型：uint8, uint256, int8 等

02 自定义值类型

核心概念：包装 (Wrap) 与展开 (Unwrap)

操作语法作用
包装 (wrap)Price.wrap(value)底层类型 → 自定义类型
展开 (unwrap)Price.unwrap(price)自定义类型 → 底层类型

为什么需要包装/展开？

  • 自定义类型本质上还是底层类型
  • 编译器会阻止不同自定义类型之间的混用（类型安全）
  • 需要计算时必须先展开为底层类型
02 自定义值类型

包装与展开示例

// 1. 创建新值（包装） function setPrice(uint256 _value) public { currentPrice = Price.wrap(_value); // uint256 → Price } // 2. 取出值计算（展开） function applyDiscount( Price _price, uint8 _discount ) public pure returns (Price) { // 展开为底层类型进行计算 uint256 rawPrice = Price.unwrap(_price); uint256 discounted = rawPrice * (100 - _discount) / 100; // 包装返回 return Price.wrap(discounted); }
02 自定义值类型

什么时候需要展开？

需要展开的场景

  • 数学运算：Price.unwrap(p1) + Price.unwrap(p2)
  • 比较操作：Price.unwrap(p1) == Price.unwrap(p2)
  • 转换为其他类型

不需要展开的场景

  • 作为映射的 key/value
  • 函数参数传递
  • 存储到状态变量
03 状态机

什么是状态机？

状态机是一种设计模式，用于管理系统只能处于有限状态之一，并在特定条件下进行状态转换。

生活中的例子

  • 电梯：停止 → 上升 → 停止 → 下降 → 停止
  • 订单：待付款 → 已付款 → 已发货 → 已收货
03 状态机

状态机的组成要素

状态机 = 状态定义 + 状态变量 + 转换规则 + 转换函数

1. 状态

用枚举定义所有可能的状态

2. 状态变量

存储当前状态

3. 转换规则

定义哪些状态可以转换到哪些状态

4. 转换函数

执行状态转换的函数

03 状态机

状态机代码示例

contract StateMachine { // 1. 定义状态枚举 enum ContractState { Draft, // 0 - 草稿 Review, // 1 - 审核中 Approved, // 2 - 已批准 Active, // 3 - 激活 Suspended, // 4 - 暂停 Terminated // 5 - 终止 } // 2. 状态变量 ContractState public state = ContractState.Draft; // 3. 修饰符：验证状态 modifier inState(ContractState _state) { require(state == _state, "Invalid state"); _; } // 4. 状态转换函数 function submitForReview() public onlyOwner inState(ContractState.Draft) { state = ContractState.Review; } }

03 状态机

状态验证方法

方法1：使用修饰符

modifier inState(ContractState _s) { require(state == _s, "Invalid"); _; } function activate() public inState(Approved) { ... }

方法2：使用 require

function terminate() public onlyOwner { require( state == Active || state == Approved, "Cannot terminate" ); state = Terminated; }

方法3：查询函数

function canTransitionTo(ContractState _target) view returns (bool) { if (state == Draft && _target == Review) return true; return false; }
