🔧 什么是构造函数？

// 使用 constructor 关键字定义构造函数 constructor(uint256 _initialValue) { owner = msg.sender; // 设置合约部署者为所有者 initialValue = _initialValue; // 初始化状态变量 }
  • 在合约部署时自动执行一次
  • 用于初始化状态变量
  • 整个合约生命周期只执行一次
  • 可以带参数，支持参数验证

📝 构造函数的类型

无参数构造函数

constructor() { owner = msg.sender; }

带参数构造函数

constructor(uint256 _value) { initialValue = _value; }

⚠️ 每个合约只能有一个构造函数

⚡ 部署字节码 vs 运行时字节码

属性 部署字节码 运行时字节码
执行时机 部署时执行一次 每次调用函数时执行
是否持久化 部署后丢弃 永久存储在区块链上
包含内容 构造函数代码 + 创建逻辑 合约所有功能代码
部署交易
→
部署字节码执行
→
运行时字节码存入链
→
合约创建成功

💎 constant 常量

在编译时就确定值的变量，值永远不会改变

// 定义常量 - 值在编译时确定 uint256 public constant MAX_SUPPLY = 10000; address public constant TREASURY = 0x1234...; string public constant TOKEN_NAME = "MyToken";
  • 值在编译时确定，不能使用运行时变量
  • 不占用存储槽，节省 Gas
  • 赋值时只能使用字面量或常量表达式

🔒 immutable 不可变量

在部署时（构造函数中）赋值一次，之后不能改变

// 定义不可变量 address public immutable owner; uint256 public immutable launchDate; // 在构造函数中赋值 - 只能在构造函数中赋值一次 constructor() { owner = msg.sender; // 部署时赋值 launchDate = block.timestamp; // 可使用运行时变量 }
  • 值在部署时（构造函数中）赋值
  • 可以使用运行时变量赋值（msg.sender、block.timestamp）
  • 不占用存储槽，节省 Gas

⚖️ constant vs immutable 对比

特性 constant immutable
赋值时机 编译时 部署时（构造函数中）
可使用运行时变量 ❌ 不可 ✅ 可以
存储位置 不占存储槽 不占存储槽
Gas 消耗 最省（内联到字节码） 较省
使用场景 值固定不变 部署时确定、之后不变

📦 字节码大小与 24KB 限制

EIP-170

24KB

合约字节码大小上限

Gas 消耗

~200 Gas

每个字节码字节

  • 超过 24KB 限制的合约无法部署
  • 字节码越大，部署成本越高
  • 优化方法：减少状态变量、使用 constant/immutable、合理函数可见性

📥 receive() 函数

专门用于接收纯以太币转账（不带任何数据）

// receive() 函数必须声明为 payable receive() external payable { // 处理收到的以太币 totalReceived += msg.value; }
transfer()
/
send()
/

→
receive()

优先匹配：当收到纯 ETH 转账时，优先调用 receive()

🔄 fallback() 函数

备用/兜底函数，处理未知函数调用和带数据的 ETH 转账

// fallback() 函数必须声明为 payable（如果要接收以太币） fallback() external payable { // 处理未知函数调用或带数据的以太币转账 totalReceived += msg.value; }
  • 调用不存在的函数
  • 调用函数但数据格式不对
  • 有数据的 ETH 转账（如果没有 receive()）

💸 发送 ETH 的三种方式

方式 语法 Gas 限制 特点
transfer to.transfer(amount) 2300 Gas 旧版，有重入风险
send to.send(amount) 2300 Gas 旧版，返回布尔值
call to.call{value: amount}("") 无限制 ✅ 新版推荐

✓ 推荐使用 call：无 Gas 限制，更安全，返回值清晰

📌 总结

  • 构造函数：合约部署时执行一次，用于初始化
  • 部署字节码：包含构造函数代码，部署后丢弃
  • 运行时字节码：永久存储在区块链上
  • constant：编译时确定，不占存储槽
  • immutable：部署时确定，不占存储槽
  • 24KB 限制：EIP-170 规定的字节码上限
  • receive()：收钱函数，只处理纯 ETH
  • fallback()：兜底函数，处理未知调用
  • call：推荐发送 ETH 方式，无 Gas 限制
