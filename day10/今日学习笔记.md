# Day10学习笔记：构造函数、字节码与以太币接收

# 1.构造函数、部署字节码与运行时字节码

## 1. 知识点介绍

### 1.1 什么是构造函数？

**构造函数（Constructor）** 是 Solidity 合约中一个特殊的函数，它在**合约部署时自动执行一次**。构造函数的主要作用是初始化合约的状态变量，设置合约的初始状态。

### 1.2 什么是部署字节码？

**部署字节码（Creation Bytecode / Deployment Bytecode）** 是用于创建合约的字节码，包含：
- 构造函数的代码，可以简单理解为用来创建合约的"安装程序"

部署字节码在合约部署时执行**一次**，部署完成后被丢弃，不再需要。

### 1.3 什么是运行时字节码？

**运行时字节码（Runtime Bytecode）** 是部署成功后永久存储在区块链上的字节码，包含：
- 合约中除构造函数之外的所有代码
- 状态变量的存储结构
- 所有函数的实现

每次调用合约函数时，执行的就是运行时字节码。

### 1.4 三者关系

```
部署交易 → 部署字节码执行（包括constructor） → 运行字节码存入区块链 → 合约创建成功
```

## 2. 构造函数详解

### 2.1 构造函数的定义

```solidity
// 使用 constructor 关键字定义构造函数
constructor() {
    // 初始化代码
}
```

### 2.2 构造函数的类型

#### 2.2.1 无参数构造函数

```solidity
// 不带任何参数的构造函数
constructor() {
    owner = msg.sender;  // 设置合约部署者为所有者
    value = 100;        // 使用默认值初始化
}
```

#### 2.2.2 带参数的构造函数

```solidity
// 带参数的构造函数
constructor(uint256 _initialValue, string memory _name) {
    require(_initialValue > 0, "Initial value must be greater than 0");
    owner = msg.sender;
    initialValue = _initialValue;
    name = _name;
}
```

#### 2.2.3 构造函数中的复杂初始化

```solidity
constructor(uint256[] memory _initialValues, address[] memory _members) {
    owner = msg.sender;

    // 初始化数组
    for (uint256 i = 0; i < _initialValues.length; i++) {
        values.push(_initialValues[i]);
    }

    // 初始化映射
    for (uint256 i = 0; i < _members.length; i++) {
        require(_members[i] != address(0), "Invalid member address");
        isMember[_members[i]] = true;
    }
}
```

### 2.3 构造函数的特点

| 特点 | 说明 |
|------|------|
| 执行时机 | 合约部署时执行一次 |
| 执行次数 | 整个合约生命周期只执行一次 |
| 可带参数 | 支持传入参数进行初始化 |
| 无返回值 | 构造函数没有返回值声明 |
| 可访问全局变量 | 可以使用 `msg.sender`、`block.timestamp` 等 |
| 唯一性 | 每个合约只能有一个构造函数 |

## 3. constant 常量与 immutable 不可变量

### 3.1 constant（常量）

**constant** 变量是在**编译时**就确定值的变量，值永远不会改变。

```solidity
// 定义常量 - 值在编译时确定
uint256 public constant MAX_SUPPLY = 10000;
address public constant TREASURY = 0x1234567890123456789012345678901234567890;
string public constant TOKEN_NAME = "MyToken";
bytes32 public constant DOMAIN_SEPARATOR = keccak256("EIP712Domain");
```

**特点**：
- 值在**编译时**确定，不能使用运行时的变量（如 block.timestamp、msg.sender）
- 不占用**存储槽**，不消耗 Gas 存储
- 节省 Gas：因为不需要从存储中读取
- 赋值时只能使用**字面量**或**常量表达式**

**注意**：以下赋值方式是**错误**的：
```solidity
// 错误！不能在 constant 中使用运行时的变量
uint256 public constant timestamp = block.timestamp;  // ❌
address public constant deployer = msg.sender;          // ❌
```

### 3.2 immutable（不可变量）

**immutable** 变量是在**部署时**（构造函数中）赋值一次，之后不能改变。

```solidity
// 定义不可变量
address public immutable owner;           // 合约所有者
address public immutable treasury;         // 国库地址
uint256 public immutable launchDate;       // 上线时间

// 在构造函数中赋值 - 只能在构造函数中赋值一次
constructor(address _treasury) {
    owner = msg.sender;                    // 部署时赋值
    treasury = _treasury;                   // 部署时赋值
    launchDate = block.timestamp;          // 部署时赋值
}
```

**特点**：
- 值在**部署时**（构造函数中）赋值一次，之后不可变
- 不占用**存储槽**，节省 Gas
- 可以使用**运行时变量**赋值（如 msg.sender、block.timestamp）
- 只能在构造函数中赋值一次

### 3.3 constant vs immutable 对比

| 特性 | constant | immutable |
|------|----------|-----------|
| 赋值时机 | 编译时 | 部署时（构造函数中） |
| 赋值方式 | 只能使用字面量或常量表达式 | 可以使用 msg.sender、block.timestamp 等 |
| 存储位置 | 不占用存储槽 | 不占用存储槽 |
| Gas 消耗 | 最省 Gas（内联到字节码中） | 较省 Gas |
| 修改 | 不可修改 | 不可修改 |
| 使用场景 | 值固定不变的场景 | 部署时确定、之后不可变的场景 |

### 3.4 使用场景示例

```solidity
// ==================== constant 使用场景 ====================

// 代币精度
uint8 public constant DECIMALS = 18;

// 代币总量
uint256 public constant TOTAL_SUPPLY = 1000000 * 10**18;

// 固定的黑名单
address public constant BURN_ADDRESS = 0x000000000000000000000000000000000000dEaD;

// 状态枚举
enum Status { Pending, Active, Completed }
Status public constant DEFAULT_STATUS = Status.Pending;

// ==================== immutable 使用场景 ====================

// 部署者地址（部署时确定）
address public immutable deployer;
constructor() {
    deployer = msg.sender;
}

// 治理合约地址（部署时传入）
address public immutable governance;
constructor(address _governance) {
    governance = _governance;
}

// 创世块时间（部署时确定）
uint256 public immutable genesisTime;
constructor() {
    genesisTime = block.timestamp;
}

// 代币精度（部署时设置，之后不变）
uint8 public immutable decimals;
constructor(uint8 _decimals) {
    decimals = _decimals;
}
```

### 3.5 Gas 优化原理

```solidity
// 普通状态变量 - 存储在链上，每次读取消耗 Gas
uint256 public value;  // 读取：SLOAD (~2100 Gas)

// constant 常量 - 不占用存储槽，编译时内联
uint256 public constant VALUE = 100;  // 读取：无 Gas 消耗（内联到字节码）

// immutable 不可变量 - 不占用存储槽，运行时从字节码读取
uint256 public immutable startTime;  // 读取：约 2100 Gas（但只需赋值一次）
constructor() {
    startTime = block.timestamp;
}
```

**Gas 节省原理**：
- 普通变量存储在 Storage 中，读取需要 SLOAD 操作（昂贵）
- constant 和 immutable 不占用 Storage 槽
- constant 在编译时内联，每次使用直接替换为常量值

## 4. 部署字节码与运行时字节码

### 3.1 概念对比

| 属性 | 部署字节码 | 运行时字节码 |
|------|------------|--------------|
| 执行时机 | 部署时执行一次 | 每次调用合约函数时执行 |
| 是否持久化 | 部署后丢弃 | 永久存储在区块链上 |
| 包含内容 | 构造函数代码 + 创建逻辑 | 合约的所有功能代码 |
| 用途 | 创建并初始化合约 | 执行合约功能 |

### 3.2 字节码查看方法

使用 Hardhat 可以查看字节码：

```javascript
// 部署字节码（包含构造函数）
console.log("Creation bytecode:", artifact.bytecode);

// 运行时字节码（不包含构造函数）
console.log("Runtime bytecode:", artifact.deployedBytecode);
```

### 3.3 字节码大小计算

```
部署字节码大小 = 运行时字节码大小 + 构造函数字节码大小
```

## 4. 字节码大小与 Gas 优化

### 4.1 字节码大小限制

**EIP-170** 规定合约字节码大小上限为 **24KB**（24576 字节）。超过此限制的合约无法部署。

### 4.2 Gas 消耗

- **每个字节码字节** 消耗约 **200 Gas**
- 部署字节码大小直接影响部署成本
- 运行时字节码大小影响存储成本

### 4.3 字节码优化方法

#### 4.3.1 减少状态变量

```solidity
// 不推荐：多个单独的状态变量
uint256 public value1;
uint256 public value2;
uint256 public value3;

// 推荐：使用 packed struct
struct PackedData {
    uint128 value1;
    uint128 value2;
}
PackedData public packedData;
```

#### 4.3.2 使用 constant 和 immutable

```solidity
// 不需要存储在链上，不增加字节码大小
uint256 public constant MAX_SUPPLY = 10000;

// 部署时赋值一次，之后不可变，比普通状态变量更省 Gas
address public immutable treasury;

constructor(address _treasury) {
    treasury = _treasury;
}
```

#### 4.3.3 合理使用函数可见性

```solidity
// 如果函数只在本合约或子合约中调用，使用 internal 比 public 更省 Gas
function _internalFunction() internal {
    // ...
}

// 如果函数只从外部调用，使用 external 比 public 更省 Gas
function externalOnly() external {
    // ...
}
```

#### 4.3.4 使用库（Library）

将可复用的代码提取到库中，避免字节码膨胀。

#### 4.3.5 启用编译器优化器

```javascript
// hardhat.config.js
solidity: {
    settings: {
        optimizer: {
            enabled: true,
            runs: 200  // 优化次数
        }
    }
}
```

### 5.1 构造函数典型应用场景

1. **权限设置**：在部署时设置合约管理员/所有者
    ```solidity
    constructor() {
        owner = msg.sender;
    ```


## 6. 示例代码

### 6.1 完整构造函数示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

/**
 * @title ConstructorDemo
 * @notice 演示 Solidity 构造函数的使用
 * @dev 用于学习构造函数的定义和使用
 *
 * 学习要点：
 * 1. 构造函数定义：constructor()
 * 2. 构造函数参数
 * 3. 构造函数中的初始化
 * 4. 构造函数执行时机
 */
contract ConstructorDemo {
    // ═══════════════════════════════════════════════════════════
    // 状态变量
    // ═══════════════════════════════════════════════════════════

    address public owner;          // 合约所有者地址
    uint256 public initialValue;   // 初始值
    string public contractName;   // 合约名称
    uint256 public deployTimestamp; // 部署时间戳

    // ═══════════════════════════════════════════════════════════
    // 构造函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice 构造函数，在合约部署时执行一次
     * @param _initialValue 初始值参数
     * @param _contractName 合约名称参数（string 类型需要 memory 修饰符）
     * @dev 构造函数在部署时执行一次，用于初始化状态变量
     *
     * 执行流程：
     * 1. 设置 owner 为部署者地址
     * 2. 设置 initialValue 为传入的初始值
     * 3. 设置 contractName 为传入的合约名称
     * 4. 记录部署时间戳
     */
    constructor(uint256 _initialValue, string memory _contractName) {
        owner = msg.sender;                    // msg.sender 是部署合约的账户地址
        initialValue = _initialValue;           // 初始化传入的值
        contractName = _contractName;           // 初始化合约名称
        deployTimestamp = block.timestamp;     // block.timestamp 是当前区块的时间戳
    }

    // ═══════════════════════════════════════════════════════════
    // 查询函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice 获取合约信息
     * @return owner 所有者地址
     * @return initialValue 初始值
     * @return contractName 合约名称
     * @return deployTimestamp 部署时间戳
     */
    function getContractInfo() public view returns (
        address owner,
        uint256 initialValue,
        string memory contractName,
        uint256 deployTimestamp
    ) {
        return (owner, initialValue, contractName, deployTimestamp);
    }
}
```

## 7. 注意事项

### 7.1 构造函数注意事项

1. **构造函数只能有一个**：Solidity 不支持函数重载，合约只能定义一个构造函数。

2. **构造函数参数传递**：部署时需要传入构造函数参数，参数类型必须与定义匹配。

3. **msg.sender 可用**：构造函数中可以使用 `msg.sender` 获取部署者地址。

4. **执行一次**：构造函数在合约创建时执行，之后无法再次调用。

5. **string 需要 memory**：构造函数参数中，string 和 bytes 类型需要使用 `memory` 修饰符。

6. **可使用 require**：构造函数中可以使用 `require` 进行参数验证。

7. **避免复杂循环**：大量循环会消耗大量 Gas，影响部署成本。

### 7.2 字节码相关注意事项

1. **24KB 限制**：合约字节码不能超过 24KB（EIP-170），否则无法部署。

2. **Gas 消耗**：字节码越大，部署成本越高。每个字节约消耗 200 Gas。

3. **public 变量生成 getter**：每个 public 状态变量会自动生成一个 getter 函数，增加字节码大小。

4. **优化器设置**：可以在 `hardhat.config.js` 中启用编译器优化器来减少字节码大小。

5. **constant 和 immutable**：使用 `constant` 和 `immutable` 修饰符可以减少运行时字节码，因为这些变量不存储在存储槽中。

# 2. receive 与 fallback 函数

## 1. 知识点介绍

### 1.1 什么是 receive() 函数？

**receive()** 是 Solidity 合约中的特殊函数，专门用于**接收纯以太币转账**（不带任何数据）。

### 1.2 什么是 fallback() 函数？

**fallback()** 不是"退回"的意思，而是"备用/兜底"函数。它用于处理以下情况：
- 调用不存在的函数
- 调用存在但数据不匹配的函数
- 有数据的以太币转账（如果没有 receive()）

### 1.3 发送 ETH 的两个层面

| 层面 | 方法 | 说明 |
|------|------|------|
| **客户端**（前端/钱包） | `sendTransaction` | 用户通过钱包发起交易 |
| **合约内部**（Solidity） | `transfer` / `send` / `call` | 合约代码内部转账 |

---

## 2. receive() 函数详解

### 2.1 定义

```solidity
// receive() 函数必须声明为 payable
receive() external payable {
    // 处理收到的以太币
}
```

### 2.2 调用场景

| 场景 | 说明 |
|------|------|
| `address.transfer(amount)` | 通过 transfer 方式转账 |
| `address.send(amount)` | 通过 send 方式转账 |
| `call{value: amount}("")` | 通过 call 方式转账（空数据） |

### 2.3 特点

- **必须声明为 payable**：否则无法接收以太币
- **只能处理纯 ETH**：不能处理任何携带的数据
- **优先匹配**：当收到纯 ETH 转账时，优先调用 receive()

---

## 3. fallback() 函数详解

### 3.1 定义

```solidity
// fallback() 函数必须声明为 payable（如果要接收以太币）
fallback() external payable {
    // 处理未知函数调用或带数据的以太币转账
}
```

### 3.2 调用场景

| 场景 | 说明 |
|------|------|
| 调用不存在的函数 | 调用的函数在合约中不存在 |
| 数据不匹配 | 调用函数但数据格式不对 |
| 有数据的 ETH 转账 | 转账时携带了 data（如果没有 receive()） |

### 3.3 特点

- **必须声明为 payable**：如果要接收以太币
- **可以处理数据**：通过 `msg.data` 获取调用数据
- **兜底处理**：当没有其他函数匹配时调用

---

## 4. 触发流程

```
发送 ETH 到合约
        ↓
   有 data 吗？
   ↙        ↘
  有          没有
   ↓          ↓
fallback()  有 receive 吗？
             ↙        ↘
            有          没有
             ↓          ↓
        receive()   ❌ revert（交易失败）
```

```
调用合约函数
        ↓
  函数存在吗？
   ↙        ↘
  是          否
   ↓          ↓
 正常执行  fallback()
```

## 5. 发送 ETH 的三种方式（合约内部）

| 方式 | 语法 | Gas 限制 | 特点 |
|------|------|---------|------|
| **transfer** | `to.transfer(amount)` | 2300 Gas | 旧版，有重入风险 |
| **send** | `to.send(amount)` | 2300 Gas | 旧版，返回布尔值 |
| **call** | `to.call{value: amount}("")` | 无限制 | **新版推荐** |

### 5.1 call 语法详解

```solidity
// 语法：<地址>.call{value: <金额>}(<数据>)

// 完整示例
(bool success, ) = to.call{value: amount}("");

// to.call      = 调用 to 地址的合约
// {value: amount} = 发送多少 ETH
// ("")         = 调用时传的 data（空字符串 = 不调用任何函数）
```

### 5.2 为什么推荐使用 call？

1. **无 Gas 限制**：transfer/send 固定 2300 Gas，可能不够用
2. **返回值清晰**：可以明确判断是否成功
3. **更安全**：避免因 Gas 不足导致的交易失败

### 7.1 完整 receive/fallback 示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

/**
 * @title ReceiveFallbackDemo
 * @notice 演示 receive 和 fallback 函数的使用
 * @dev 用于学习如何让合约能接收以太币
 *
 * 学习要点：
 * 1. receive() 函数：接收纯以太币转账
 * 2. fallback() 函数：处理未知函数调用和以太币转账
 * 3. 通过 send 或 transfer 方式向合约发送 ETH
 * 4. 对 ICO、钱包合约、捐赠合约等场景的应用
 */
contract ReceiveFallbackDemo {
    // ═══════════════════════════════════════════════════════════
    // 状态变量
    // ═══════════════════════════════════════════════════════════

    address public owner;           // 合约所有者
    uint256 public totalReceived;   // 总接收金额
    uint256 public receiveCount;    // receive 函数调用次数
    uint256 public fallbackCount;   // fallback 函数调用次数

    mapping(address => uint256) public balances;  // 地址余额映射
    address[] public contributors;               // 贡献者列表

    // ═══════════════════════════════════════════════════════════
    // 事件
    // ═══════════════════════════════════════════════════════════

    event Received(address indexed sender, uint256 amount);  // 收到 ETH 事件
    event FallbackCalled(address indexed sender, uint256 amount, bytes data);  // fallback 调用事件
    event Withdrawn(address indexed to, uint256 amount);   // 提现事件

    // ═══════════════════════════════════════════════════════════
    // 构造函数
    // ═══════════════════════════════════════════════════════════

    constructor() {
        owner = msg.sender;  // 设置部署者为所有者
    }

    // ═══════════════════════════════════════════════════════════
    // receive() 函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice receive() 函数 - 接收纯以太币转账（没有数据）
     * @dev 当收到纯 ETH 转账时自动调用
     *
     * 调用场景：
     * - address.transfer(amount) 或 address.send(amount)
     * - call{value: amount}("")
     *
     * 注意：
     * - receive() 必须声明为 payable
     * - 如果没有 receive()，会尝试 fallback()
     * - receive() 只能处理纯 ETH，不能处理携带的数据
     */
    receive() external payable {
        // 增加总接收金额
        totalReceived += msg.value;
        // 增加 receive 调用计数
        receiveCount++;

        // 记录贡献者（首次捐赠时添加到列表）
        if (balances[msg.sender] == 0) {
            contributors.push(msg.sender);
        }
        // 更新贡献者余额
        balances[msg.sender] += msg.value;

        // 触发事件
        emit Received(msg.sender, msg.value);
    }

    // ═══════════════════════════════════════════════════════════
    // fallback() 函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice fallback() 函数 - 处理未知函数调用和以太币转账
     * @dev 当没有其他函数匹配时调用
     *
     * 调用场景：
     * - 调用不存在的函数
     * - 调用存在但数据不匹配的函数
     * - 有数据的以太币转账（如果没有 receive()）
     *
     * 注意：
     * - fallback() 必须声明为 payable（如果要接收以太币）
     * - fallback() 可以接收数据参数（通过 msg.data）
     * - fallback() 是兜底处理，可以同时处理 ETH 和数据
     */
    fallback() external payable {
        // 增加总接收金额
        totalReceived += msg.value;
        // 增加 fallback 调用计数
        fallbackCount++;

        // 记录贡献者
        if (balances[msg.sender] == 0) {
            contributors.push(msg.sender);
        }
        // 更新贡献者余额
        balances[msg.sender] += msg.value;

        // 触发事件（包含发送的数据）
        emit FallbackCalled(msg.sender, msg.value, msg.data);
    }

    // ═══════════════════════════════════════════════════════════
    // 普通函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice 获取合约余额
     * @return 合约当前余额
     */
    function getBalance() public view returns (uint256) {
        return address(this).balance;
    }

    /**
     * @notice 获取贡献者数量
     * @return 贡献者总数
     */
    function getContributorCount() public view returns (uint256) {
        return contributors.length;
    }

    /**
     * @notice 提取以太币（只有 owner 可以调用）
     * @param to 接收地址（必须是 payable）
     * @param amount 提取数量
     */
    function withdraw(address payable to, uint256 amount) public {
        require(msg.sender == owner, "Only owner");           // 仅所有者可调用
        require(address(this).balance >= amount, "Insufficient balance");  // 余额足够

        // 使用 call 方式）
        (bool success, ) = to.call{发送 ETH（推荐value: amount}("");
        require(success, "Transfer failed");

        emit Withdrawn(to, amount);
    }

    /**
     * @notice 提取所有以太币（只有 owner 可以调用）
     * @param to 接收地址
     */
    function withdrawAll(address payable to) public {
        require(msg.sender == owner, "Only owner");
        uint256 balance = address(this).balance;
        require(balance > 0, "No balance");

        (bool success, ) = to.call{value: balance}("");
        require(success, "Transfer failed");

        emit Withdrawn(to, balance);
    }
}
```

### 7.2 捐赠合约示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

/**
 * @title DonationContract
 * @notice 捐赠合约示例
 * @dev 演示 receive 和 fallback 在捐赠场景中的应用
 *
 * 学习要点：
 * 1. 使用 receive() 接收捐赠
 * 2. 记录捐赠者信息
 * 3. 提取捐赠资金
 * 4. 对 ICO、钱包合约、捐赠合约等场景的应用
 */
contract DonationContract {
    // ═══════════════════════════════════════════════════════════
    // 状态变量
    // ═══════════════════════════════════════════════════════════

    address public owner;           // 合约所有者
    address public beneficiary;      // 受益人地址

    // 捐赠记录结构体
    struct Donation {
        address donor;     // 捐赠者地址
        uint256 amount;    // 捐赠金额
        uint256 timestamp; // 捐赠时间戳
        string message;    // 捐赠消息
    }

    Donation[] public donations;                     // 所有捐赠记录
    mapping(address => uint256) public totalDonated; // 地址 -> 总捐赠额
    mapping(address => uint256[]) public donorDonationIndices;  // 地址 -> 捐赠索引列表

    uint256 public totalDonations;  // 捐赠总额
    uint256 public donationCount;   // 捐赠总数

    // ═══════════════════════════════════════════════════════════
    // 事件
    // ═══════════════════════════════════════════════════════════

    event DonationReceived(address indexed donor, uint256 amount, uint256 timestamp, string message);
    event FundsWithdrawn(address indexed to, uint256 amount);
    event BeneficiaryChanged(address indexed oldBeneficiary, address indexed newBeneficiary);

    // ═══════════════════════════════════════════════════════════
    // 构造函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice 构造函数
     * @param _beneficiary 受益人地址
     */
    constructor(address _beneficiary) {
        require(_beneficiary != address(0), "Invalid beneficiary");  // 验证受益人地址
        owner = msg.sender;
        beneficiary = _beneficiary;
    }

    // ═══════════════════════════════════════════════════════════
    // receive() 函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice receive() 函数 - 接收纯以太币转账（没有数据）
     * @dev 用于接收纯 ETH 捐赠
     *
     * 使用场景：
     * - 通过 send 或 transfer 方式向合约发送 ETH
     * - 直接转账捐赠
     */
    receive() external payable {
        require(msg.value > 0, "Donation amount must be greater than 0");

        // 记录捐赠
        donations.push(Donation({
            donor: msg.sender,
            amount: msg.value,
            timestamp: block.timestamp,
            message: ""
        }));

        // 更新捐赠统计
        totalDonated[msg.sender] += msg.value;
        donorDonationIndices[msg.sender].push(donationCount);
        totalDonations += msg.value;
        donationCount++;

        emit DonationReceived(msg.sender, msg.value, block.timestamp, "");
    }

    // ═══════════════════════════════════════════════════════════
    // fallback() 函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice fallback() 函数 - 处理带数据的以太币转账
     * @dev 用于处理带数据的捐赠
     *
     * 使用场景：
     * - 带消息的捐赠
     * - 未知函数调用
     */
    fallback() external payable {
        require(msg.value > 0, "Donation amount must be greater than 0");

        // 尝试解析消息（如果有）
        string memory message = "";
        if (msg.data.length > 0) {
            // 简单示例：将数据转换为字符串
            message = "Data received";
        }

        // 记录捐赠
        donations.push(Donation({
            donor: msg.sender,
            amount: msg.value,
            timestamp: block.timestamp,
            message: message
        }));

        // 更新捐赠统计
        totalDonated[msg.sender] += msg.value;
        donorDonationIndices[msg.sender].push(donationCount);
        totalDonations += msg.value;
        donationCount++;

        emit DonationReceived(msg.sender, msg.value, block.timestamp, message);
    }

    // ═══════════════════════════════════════════════════════════
    // 普通函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice 捐赠函数（带消息）
     * @param message 捐赠消息
     */
    function donate(string memory message) public payable {
        require(msg.value > 0, "Donation amount must be greater than 0");

        donations.push(Donation({
            donor: msg.sender,
            amount: msg.value,
            timestamp: block.timestamp,
            message: message
        }));

        totalDonated[msg.sender] += msg.value;
        donorDonationIndices[msg.sender].push(donationCount);
        totalDonations += msg.value;
        donationCount++;

        emit DonationReceived(msg.sender, msg.value, block.timestamp, message);
    }

    /**
     * @notice 提取资金到受益人地址
     */
    function withdraw() public {
        // 只有 owner 或 beneficiary 可以提现
        require(msg.sender == owner || msg.sender == beneficiary, "Not authorized");
        uint256 balance = address(this).balance;
        require(balance > 0, "No funds to withdraw");

        // 使用 call 发送 ETH（推荐）
        (bool success, ) = payable(beneficiary).call{value: balance}("");
        require(success, "Transfer failed");

        emit FundsWithdrawn(beneficiary, balance);
    }

    /**
     * @notice 设置受益人（只有 owner 可以调用）
     * @param _beneficiary 新受益人地址
     */
    function setBeneficiary(address _beneficiary) public {
        require(msg.sender == owner, "Only owner");
        require(_beneficiary != address(0), "Invalid beneficiary");

        address oldBeneficiary = beneficiary;
        beneficiary = _beneficiary;

        emit BeneficiaryChanged(oldBeneficiary, _beneficiary);
    }

    /**
     * @notice 获取捐赠信息
     * @param index 捐赠索引
     * @return donor 捐赠者地址
     * @return amount 捐赠金额
     * @return timestamp 捐赠时间戳
     * @return message 捐赠消息
     */
    function getDonation(uint256 index) public view returns (
        address donor,
        uint256 amount,
        uint256 timestamp,
        string memory message
    ) {
        require(index < donations.length, "Index out of bounds");
        Donation memory donation = donations[index];
        return (donation.donor, donation.amount, donation.timestamp, donation.message);
    }

    /**
     * @notice 获取合约余额
     */
    function getBalance() public view returns (uint256) {
        return address(this).balance;
    }
}
```

### 7.3 钱包合约示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

/**
 * @title WalletContract
 * @notice 钱包合约示例
 * @dev 演示 receive 和 fallback 在钱包场景中的应用
 *
 * 学习要点：
 * 1. 使用 receive() 接收以太币
 * 2. 多签钱包功能
 * 3. 提取资金功能
 */
contract WalletContract {
    // ═══════════════════════════════════════════════════════════
    // 状态变量
    // ═══════════════════════════════════════════════════════════

    address public owner;                              // 合约所有者
    mapping(address => bool) public authorized;        // 授权地址映射
    address[] public authorizedList;                   // 授权地址列表

    uint256 public requiredApprovals;                  // 需要的批准数量
    mapping(bytes32 => bool) public approvedTransactions;    // 已批准的交易
    mapping(bytes32 => uint256) public transactionApprovalCount;  // 交易批准计数

    // ═══════════════════════════════════════════════════════════
    // 事件
    // ═══════════════════════════════════════════════════════════

    event Received(address indexed sender, uint256 amount);
    event Authorized(address indexed account);
    event Unauthorized(address indexed account);
    event TransactionProposed(bytes32 indexed txHash, address indexed to, uint256 amount);
    event TransactionApproved(bytes32 indexed txHash, address indexed approver);
    event TransactionExecuted(bytes32 indexed txHash, address indexed to, uint256 amount);

    // ═══════════════════════════════════════════════════════════
    // 构造函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice 构造函数
     * @param _authorized 授权地址数组
     * @param _requiredApprovals 需要的最少批准数
     */
    constructor(address[] memory _authorized, uint256 _requiredApprovals) {
        require(_authorized.length > 0, "At least one authorized address required");
        require(_requiredApprovals > 0 && _requiredApprovals <= _authorized.length, "Invalid required approvals");

        owner = msg.sender;
        requiredApprovals = _requiredApprovals;

        // 初始化授权地址
        for (uint256 i = 0; i < _authorized.length; i++) {
            require(_authorized[i] != address(0), "Invalid address");
            authorized[_authorized[i]] = true;
            authorizedList.push(_authorized[i]);
        }
    }

    // ═══════════════════════════════════════════════════════════
    // receive() 函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice receive() 函数 - 接收纯以太币转账
     * @dev 钱包接收资金
     */
    receive() external payable {
        emit Received(msg.sender, msg.value);
    }

    // ═══════════════════════════════════════════════════════════
    // fallback() 函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice fallback() 函数 - 处理未知函数调用和以太币转账
     */
    fallback() external payable {
        emit Received(msg.sender, msg.value);
    }

    // ═══════════════════════════════════════════════════════════
    // 普通函数
    // ═══════════════════════════════════════════════════════════

    /**
     * @notice 添加授权地址（只有 owner 可以调用）
     * @param account 要授权的地址
     */
    function addAuthorized(address account) public {
        require(msg.sender == owner, "Only owner");
        require(account != address(0), "Invalid address");
        require(!authorized[account], "Already authorized");

        authorized[account] = true;
        authorizedList.push(account);

        emit Authorized(account);
    }

    /**
     * @notice 移除授权地址（只有 owner 可以调用）
     * @param account 要移除的地址
     */
    function removeAuthorized(address account) public {
        require(msg.sender == owner, "Only owner");
        require(authorized[account], "Not authorized");

        authorized[account] = false;

        emit Unauthorized(account);
    }

    /**
     * @notice 提议交易
     * @param to 接收地址
     * @param amount 转账金额
     * @return txHash 交易哈希
     */
    function proposeTransaction(address payable to, uint256 amount) public returns (bytes32) {
        require(authorized[msg.sender], "Not authorized");
        require(to != address(0), "Invalid address");
        require(amount > 0, "Amount must be greater than 0");
        require(address(this).balance >= amount, "Insufficient balance");

        bytes32 txHash = keccak256(abi.encodePacked(to, amount, block.timestamp));

        emit TransactionProposed(txHash, to, amount);

        return txHash;
    }

    /**
     * @notice 批准交易
     * @param txHash 交易哈希
     */
    function approveTransaction(bytes32 txHash) public {
        require(authorized[msg.sender], "Not authorized");
        require(!approvedTransactions[txHash], "Already approved");

        approvedTransactions[txHash] = true;
        transactionApprovalCount[txHash]++;

        emit TransactionApproved(txHash, msg.sender);
    }

    /**
     * @notice 执行交易（需要足够的批准）
     * @param to 接收地址
     * @param amount 转账金额
     */
    function executeTransaction(address payable to, uint256 amount) public {
        require(authorized[msg.sender], "Not authorized");
        require(to != address(0), "Invalid address");
        require(amount > 0, "Amount must be greater than 0");
        require(address(this).balance >= amount, "Insufficient balance");

        bytes32 txHash = keccak256(abi.encodePacked(to, amount, block.timestamp));
        require(transactionApprovalCount[txHash] >= requiredApprovals, "Insufficient approvals");

        // 使用 call 发送 ETH（推荐）
        (bool success, ) = to.call{value: amount}("");
        require(success, "Transfer failed");

        emit TransactionExecuted(txHash, to, amount);
    }

    /**
     * @notice 获取合约余额
     */
    function getBalance() public view returns (uint256) {
        return address(this).balance;
    }
}
```

## 8. 注意事项

### 8.1 receive() 注意事项

1. **必须声明 payable**：否则无法接收以太币
2. **不能处理数据**：只能接收纯 ETH
3. **优先匹配**：当有 receive() 时，优先调用 receive()

### 8.2 fallback() 注意事项

1. **必须声明 payable**：如果要接收以太币
2. **可以处理数据**：通过 `msg.data` 获取
3. **兜底作用**：当没有其他函数匹配时调用
4. **避免复杂逻辑**：不要在 fallback 中做过多的操作

### 8.3 发送 ETH 注意事项

1. **推荐使用 call**：无 Gas 限制，更安全
2. **检查返回值**：always check return value
3. **防止重入攻击**：使用 Checks-Effects-Interactions 模式
4. **两个层面区分**：
   - `sendTransaction` 是**客户端方法**（前端/钱包）
   - `transfer/send/call` 是**合约内部方法**

### 8.4 如果没有 receive/fallback 会怎样？

| 合约情况 | 收到 ETH 的结果 |
|----------|----------------|
| 有 receive() | ✅ 成功接收 |
| 有 fallback() | ✅ 成功接收 |
| 只有普通函数 | ❌ 交易失败，ETH 退回 |
| 什么函数都没有 | ❌ 交易失败，ETH 退回 |

## 9. 总结

### Day 11 总结

1. **构造函数**：合约部署时执行一次的特殊函数，用于初始化状态变量
2. **部署字节码**：包含构造函数代码的字节码，部署后丢弃
3. **运行时字节码**：永久存储在区块链上的合约代码
4. **字节码优化**：通过各种方法减小字节码大小，降低 Gas 成本
5. **24KB 限制**：EIP-170 规定的合约字节码大小上限

### Day 12 总结

1. **receive()**：专门收钱的函数，只处理纯 ETH（无 data）
2. **fallback()**：备用/兜底函数，处理未知函数调用和带数据的 ETH 转账
3. **发送 ETH 两个层面**：
   - 客户端：`sendTransaction`
   - 合约内部：`transfer` / `send` / `call`
4. **推荐使用 call**：无 Gas 限制，更安全
5. **应用场景**：ICO、钱包合约、捐赠合约等
